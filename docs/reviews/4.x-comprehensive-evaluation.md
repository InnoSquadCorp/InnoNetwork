## 0. 한 줄 요약

**4.0.0은 "Apple-only · Swift 6 strict concurrency · 단일 transport 진입점 + 명시적 정책 합성"을 핵심 정체성으로 잡은 *프로덕션 그레이드 베이스라인*이다.** v1 식별 P1 7건 중 핵심 6건(P1.1 CircuitBreaker 4xx 카운터 / P1.7 streaming buffering / P1.12 closeHandshake 설정화 / P1.14 매크로 SourceLocation / 그리고 P1.4·P1.5·P1.8 재평가 후 *기존 구현이 이미 안전*으로 판정)이 PR #36에서 머지되었고, **P2 가장 무거운 항목인 WebSocket lifecycle reducer는 PR #37에서 generation/attempt/manual-disconnect/close-disposition payload를 묶은 package-internal FSM 도입으로 클로즈**되었다. **잔여 P1 4건(P1.2 304-with-new-Vary 보존 · P1.10 DownloadState 전환 가드 회귀 테스트 · P1.11 URLError 매핑 contract lock · P1.13 POST 503 + Retry-After + Idempotency-Key 통합)과 P2 4건(P2.1 refresh-aware coalescer lane segregation · P2.2 Interceptor 시맨틱 docstring · P2.3 DownloadManager.shared 제거 · P2.5 DecodingInterceptorCookbook)은 4.0.0 follow-up(`chore/4.1.x-followup`, PR #38)에서 모두 closed.** 동일 라인에서 P3 cookbook quartet(WebSocketProtocolPolicy / WebSocketBackgroundTransition / ObservabilityExporters / OpenAPIGeneratorRecipe), Persistent Response Cache Product RFC, Alamofire 마이그레이션 비교, WebSocketLifecycleReducer property-based fuzz도 함께 출하.

차기 메이저 후보는 Refresh lifecycle reducer 확장, Download side-effect ownership 일반화, persistent response cache 운영 정책 hardening, public Generic Execution Policy, NIO transport adapter, WebSocket `permessage-deflate`(RFC 7692)이다.

## 1.3 PR #37 변경 사항 요약 (WebSocket FSM reducer, 2026-05-01)

**브랜치**: `codex/innonetwork-websocket-fsm` · **상태**: MERGED 가정 · **+1489/-263 lines, 15 files** · 3 commits.

### 1.3.1 핵심 변경

PR #35의 stale `didOpen` 분기 패치(§10.5.2)는 *해당 race를 차단*했지만, 부수효과 명세가 여전히 분산돼 있었다. PR #37은 WebSocket lifecycle을 **package-internal reducer/FSM**으로 승격해 "결정은 순수, 효과는 ordered list"로 분리한다.

| 컴포넌트 | 위치 | 역할 |
|---|---|---|
| `WebSocketLifecycleState` | `Sources/InnoNetworkWebSocket/WebSocketLifecycleReducer.swift` | 7-case ADT × generation/attempt/autoReconnect/manualDisconnect/closeCode/disposition/error payload. `publicState: WebSocketState` projection으로 public 표면 안정 유지. |
| `WebSocketLifecycleEvent` | 동일 | `connect / didOpen / manualDisconnect / didClose / failure / closeTimeout / reconnectTimerFired / reset` 8 case. |
| `WebSocketLifecycleEffect` | 동일 | `startConnection / startHeartbeat / cancelHeartbeat / cancelReconnect / cancelMessageListener / cleanupRuntime / publishEvent / scheduleReconnect / finishStream / removeRegistry / ignoreStaleCallback` — ordered list. |
| `WebSocketLifecycleReducer.reduce(state, event, context)` | 동일 | 순수 함수. `WebSocketLifecycleDecisionContext`(reconnectAction + attempt)를 외부에서 주입해 reducer 자체는 시간/I/O 효과 없음. |
| `WebSocketTask.applyLifecycleEvent(_:context:)` | `WebSocketTask.swift` | reducer 호출 + 내부 상태 갱신 + `(state, [Effect])` 반환. |
| `WebSocketTask.restoreStateForTesting(_:)` | 동일 | 테스트 전용 직접 상태 주입 surface. 프로덕션 `updateState`는 legal transition 강제. |
| `WebSocketRuntimeRegistry.identifierToGeneration` | `WebSocketRuntimeRegistry.swift` | URLSession task identifier → connection generation 매핑. delegate callback이 stale 여부를 generation match로 판정. |
| `WebSocketManager.executeLifecycleEffects` | `WebSocketManager.swift` | reducer가 반환한 effect list를 ordered 실행. terminal cleanup은 generation-checked finalizer 단일 경로. |

### 1.3.2 닫힌 평가 항목

|---|---|

### 1.3.3 테스트 변경

- `Tests/InnoNetworkWebSocketTests/WebSocketLifecycleReducerTests.swift` 신규 — reducer pure 함수 단위 테스트 (state×event 매트릭스 + ordered effect 검증).

### 1.3.4 동반 문서 갱신

- `docs/ImprovementBacklog.md` — "Internal WebSocket Lifecycle Reducer" P2 항목을 *Completed*로 이동, generation-scoped stale callback 항목 추가.

## 2. 강점

### 2.1 API 설계
- **단일 transport 진입점**(4.0 핵심 결정): `contentType`/`requestEncoder`/`queryEncoder`/`queryRootKey`/`decoder`/`responseDecoder` 6개 프로퍼티 → `transport: TransportPolicy<APIResponse>` 1개로 통합. method-aware 기본값(`GET → .query()`, 그 외 `.json()`).
- **fluent `Endpoint`**(`Endpoint.swift:204-246`): chainable `.transport()`, `.headers()`, `.parameters()`, `.decoding()`. `.none` 명시 시 자동 promotion 안 됨 — 사용자 의도 보존 (`05e9d82` 커밋).
- **stable contract 분리**: `API_STABILITY.md` + `Scripts/api_public_symbols.allowlist`(900+줄) + `Scripts/check_docs_contract_sync.sh` — symbol-level 게이트.
- **safeDefaults**: 과도한 기본값을 안전 쪽으로 묶음(opt-in resilience policy + 보수적 retry default).
- **`responseBodyLimit` opt-in 가드** *(PR #36)*: 의도치 않은 거대 응답이 메모리를 점유하는 경로 차단.

### 2.2 Concurrency / Sendability
- **Swift 6 strict mode** 패키지 전체 강제(`swiftLanguageMode(.v6)`).
- **`final class DefaultNetworkClient: Sendable`**: isolation hop 없이 actor 호출 회피.
- **모든 정책 Sendable 전수화** + `SendableUnderlyingError`로 URLError 보존.
- **상태 머신은 actor 또는 isolated struct**: `DownloadTask`(actor), `CircuitBreakerHostState`(actor 내), `WebSocketLifecycleState`(actor 내, **reducer-driven · PR #37**).
- **WebSocket reducer pure function** *(PR #37)*: 결정은 `(state, event, context) -> (state, [Effect])` — 시간/I/O 효과 격리, 단위 테스트 결정적.

### 2.3 운영 가시성
- **`NetworkEventHub`**(partitioned event broker) + **`EventPipelineMetricsProxy`**: latency/error/byte 카운터, publish 시맨틱 명시 (PR #36 docstring).
- **`DefaultNetworkLogger`** + `OSLog` 통합, redaction 기본 ON.
- **WebSocket 하트비트 이벤트**: `.ping`/`.pong`/`.error(.pingTimeout)` 안정 공개.
- **DownloadManager append-log 이벤트소싱 persistence**: 크래시 복구 가능.

### 2.4 보안
- **Public-key pinning** with `unionAllMatches` / `mostSpecificHost` 전략 명시.
- **`Ed25519` SPKI pinning** 지원.
- **다운로드 파일 접근 권한**: 기본 0o600, fsync 정책 explicit.
- **multipart 페이로드 redaction default**.

### 2.5 품질 인프라
- **TSAN nightly**(`.github/workflows/tsan.yml`).
- **DocC + allowlist 동기 게이트**(`check_docs_contract_sync.sh`).

## 10.5 WebSocket lifecycle race — PR #35 부분 + PR #37 reducer 완전 해결

### 10.5.1 v1 시점에 발견된 race

수동 `disconnect(task)` 진행 중 `.disconnecting` 상태에서 늦게 도착한 stale `didOpen` 콜백이 close-handshake timeout을 cancel하거나 runtime registry를 부분 정리하는 경로가 있어 — terminal cleanup이 partial 상태로 남아 `.disconnected` 발행이 영구 지연될 수 있었음.

### 10.5.2 PR #35 부분 해결

`WebSocketManager`에 `state == .disconnecting` 별도 분기를 추가해 stale `didOpen` 콜백이 registry 정리 없이 즉시 return하도록 함. close-handshake timeout 또는 `didClose`가 단독으로 terminal cleanup 완수.

### 10.5.3 PR #37 reducer 일반화 (완전 해결)

PR #35 분기 패치를 reducer로 일반화:

- `WebSocketLifecycleReducer.reduce(state, event, context)`가 *모든* lifecycle 이벤트(connect / didOpen / manualDisconnect / didClose / failure / closeTimeout / reconnectTimerFired / reset)를 단일 결정 함수로 통합.
- delegate callback의 stale 여부는 `WebSocketRuntimeRegistry.identifierToGeneration` 매핑으로 generation 비교 — mismatch 시 reducer가 `ignoreStaleCallback` 단일 effect로 reduce.
- terminal cleanup의 4 effect(runtime cancel → publish event → finish stream → registry remove)를 reducer가 ordered 반환, manager는 기계적으로 실행. *경로 한 곳*에서만 발생.

### 10.5.4 동반 테스트

- `WebSocketLifecycleReducerTests` (PR #37 신규): reducer pure function 단위 테스트. state×event 매트릭스 + ordered effect 검증.
- `WebSocketTests` *Stale close callback does not advance reconnect attempts* (PR #37 신규): generation 증가 후 stale `handleDisconnected` → `.connected` 유지 + reconnect attempt 증가 0 + 신규 URLSession task 생성 0.
- PR #35의 deterministic invariant 5종(`did-open-after-disconnect`, `terminal cleanup atomicity`, `manual-disconnect-wins`, `event hub finish`, `registry removal ordering`) 그대로 유지.

### 10.5.5 본 평가에서 v1에 식별 못 한 이유 + 후속 일반화 후보

본 평가 v1은 `WebSocketManager`를 정적으로만 읽었고 동시성 시뮬레이션이 부재했음 → race 누락. PR #35의 deterministic invariant 테스트가 race를 노출, PR #37이 reducer로 패턴 일반화. **다음 단계**: 같은 reducer 패턴을 `DownloadManager` cancel-during-restore-after-failure / `RequestExecutor` refresh-during-circuit-open / WebSocket reconnect-during-disconnecting 등에 적용 검토 (§3.3 P3).
