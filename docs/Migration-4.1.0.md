# Historical 4.1.0 Migration Draft

No `4.1.0` tag was published. This draft described changes that were already
present in the public `4.0.0` tag, so it is not an upgrade step between 4.0
and 5.0 and must not be applied as a separate migration.

Use [Migration-4.0.0.md](Migration-4.0.0.md) for the original public baseline
and [Migration-5.0.0.md](Migration-5.0.0.md) for every change required when
moving from 4.x to 5.0.
