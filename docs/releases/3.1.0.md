# InnoNetwork 3.1.0 Release Notes

## Summary

InnoNetwork 3.1.0 is the current public OSS release line for the framework family:

- `InnoNetwork`
- `InnoNetworkDownload`
- `InnoNetworkWebSocket`

This minor release keeps the `3.x` line source-compatible while adding a supported
low-level typed execution path for higher networking layers.

## Highlights

### Low-level execution extension point

- Added `NetworkClient.perform(_:)` as the public low-level typed execution API
- Promoted `SingleRequestExecutable` to a supported public contract for framework
  and policy-layer integrations
- Kept `request(_:)` and `upload(_:)` as the recommended high-level APIs for most consumers

### Better layering support

- Higher-level packages can now adapt custom request contracts onto `InnoNetwork`
  without relying on SPI imports
- The low-level execution path still delegates retry coordination, trust handling,
  request building, and observability to `InnoNetwork`

### Documentation and stability guidance

- Added explicit README and DocC guidance for when to use `perform(_:)`
- Updated `API_STABILITY.md` so `request(_:)` and `upload(_:)` stay stable while
  `perform(_:)` and `SingleRequestExecutable` are documented as provisionally stable
- Kept docs-contract-sync aligned with the new release contract

## Upgrade Notes

- Package installation instructions now target `3.1.0`
- Existing `request(_:)` and `upload(_:)` integrations continue to work without source changes
- Integrations that previously relied on SPI request execution can migrate to the
  new public `perform(_:)` entry point

## Validation

Release validation for 3.1.0 includes:

- `swift test`

- benchmark target smoke build and quick benchmark run
- external consumer smoke build
