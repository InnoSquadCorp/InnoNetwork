# InnoNetwork 4.1.0

> Risk-mitigation epic on top of 4.0. Purely additive on the public API
> surface — no breaking changes, no deprecations of stable 4.0 APIs.

## Highlights

- **Per-endpoint `acceptableStatusCodes` override** so an endpoint can
  treat `304 Not Modified` (or any other code) as success without
  changing the session-wide default.
- **WebSocket reconnect counters split** into `attemptedReconnectCount`
  (per cycle, includes overshoot) and `successfulReconnectCount`
  (cumulative). The legacy `reconnectCount` is preserved as a
  `@deprecated` alias.
- **`DownloadManager.shared` no longer crashes** on duplicate session
  identifiers — it logs an OSLog `.fault`, asserts in DEBUG, and falls
  back to a process-unique identifier. Use the new
  `DownloadManager.make(configuration:)` throwing factory when explicit
  failure handling is required.
- **`StreamingResumePolicy.lastEventID`** drives optional Last-Event-ID
  reconnect on streaming endpoints, gated behind a per-endpoint hook
  so existing streams keep their previous semantics.
- **`MultipartUploadStrategy`** (opt-in `.streamingThreshold(bytes:)` /
  `.alwaysStream`) bounds peak memory for large multipart uploads by
  spooling the body to a temp file and using
  `URLSession.upload(for:fromFile:)`.
- **`NetworkError` failure-payload redaction by default**.
  `Response.data` attached to `objectMapping`, `jsonMapping`,
  `statusCode`, and `underlying` cases is zeroed out unless
  `NetworkConfiguration.captureFailurePayload = true`. PII can no
  longer leak into crash logs through the error chain.
- **WebSocket bounded send queue** with `.fail` / `.dropNewest` overflow
  policy. Caps per-task in-flight send concurrency so a runaway
  producer cannot pile unbounded operations onto a slow socket.
- **DocC site is live** at https://innosquadcorp.github.io/InnoNetwork/
  with onboarding articles for retry decisions, error classification,
  trust policies, background downloads, persistence, close codes, and
  reconnect.
- **Production Checklist** section in the README and Korean mirror at
  `docs/ko/README.md`.
- **Live-endpoint nightly smoke** (httpbin / ws.postman-echo) gated
  behind `INNO_LIVE=1`. Default `swift test` keeps the cases skipped.
- **Coverage artifact + tightened benchmark gate** (50% → 10%) in CI.
- **CycloneDX SBOM and sigstore cosign keyless signing** for tagged
  release artifacts. `SECURITY.md` documents the `cosign verify-blob`
  invocation.

## Migration shortcuts

The 4.1 line is API-additive; the only behaviour change to be aware of
is the failure-payload redaction default.

| Pre-4.1 behaviour | 4.1 behaviour | Recovery |
|-------------------|---------------|----------|
| `NetworkError.objectMapping(_, response).response.data` had the failing response body. | `response.data` is `Data()`. Status code, headers, and request remain. | Set `NetworkConfiguration.advanced { $0.captureFailurePayload = true }` for diagnostic builds. |
| `WebSocketTask.reconnectCount` was the sole counter. | Same value reads still compile; the property is deprecated. | Migrate to `attemptedReconnectCount` (per cycle) or `successfulReconnectCount` (cumulative) per your dashboard need. |

## Deferred for follow-up

- `DownloadManager` actor refactor (R6) and async append-log persistence
  with explicit fsync (R7) — tracked in [ROADMAP.md](../ROADMAP.md).
- WebSocket `permessage-deflate` (RFC 7692) — requires substituting the
  transport because `URLSessionWebSocketTask` does not expose deflate
  negotiation. Tracked for v5.

## Verification

```bash
sudo xcode-select -s /Applications/Xcode_26.0.1.app
xcrun swift package resolve
xcrun swift build
xcrun swift test --parallel --enable-code-coverage

# Optional: live-endpoint smoke
INNO_LIVE=1 xcrun swift test --filter InnoNetworkLiveTests

# Optional: benchmark gate
xcrun swift run InnoNetworkBenchmarks --quick \
  --enforce-baseline \
  --max-regression-percent 10
```
