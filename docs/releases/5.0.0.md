# InnoNetwork 5.0.0 Release Notes

Status: Draft (unreleased)

Release date: TBD

InnoNetwork 5.0 is a planned breaking release that makes endpoint
authentication, wire-request identity, transport admission, and release
integrity explicit. It keeps the Apple platform floors at iOS 16, macOS 14,
tvOS 16, watchOS 9, and visionOS 1, and requires Swift 6.2. This draft is not a
published release and must not be used to create a `5.0.0` tag.

Read [`docs/Migration-5.0.0.md`](../Migration-5.0.0.md) before upgrading from
4.x.

## Breaking changes

- Type-level `AuthScope`, `PublicAuthScope`, and `AuthRequiredScope`, plus the
  macro-only `APIAuthentication`, are replaced by the value-level
  `SessionAuthentication`. Manual buffered, multipart, streaming, and OpenAPI
  endpoint shapes must expose the witness, while every macro endpoint must
  select `.anonymous`, `.optional`, or `.required` at the attribute.
  `EndpointBuilder<Response, Scope>` becomes `EndpointBuilder<Response>` with
  an `authentication:` initializer argument or `.authentication(_:)` modifier.
- `HTTPMethod` changes from a closed five-case enum to a validated, extensible
  `RawRepresentable` struct. Exhaustive switches over the old enum must become
  equality/raw-value checks with an explicit fallback. Standard constants now
  include `HEAD`, `CONNECT`, `OPTIONS`, and `TRACE`; valid extension-method
  tokens can be created with the failable `init(rawValue:)`.
- `RequestExecutionNext.execute(_:)` is replaced by `execute()`. Execution
  policies can observe, short-circuit, or replay the request captured by the
  executor; request mutation belongs in `RequestInterceptor`.
- The seven deprecated `NetworkConfiguration.with(...)` methods are removed.
  Use named configuration packs with `NetworkConfiguration.advanced(...)`.
- `NetworkConfiguration.recommendedForProduction(baseURL:)` is removed. There
  is no server-independent production retry, circuit-breaker, or idempotency
  policy; start with `safeDefaults(baseURL:)` and opt into reviewed policies
  through `advanced(...)`.
- Configuration packs expose named initializers but no longer publish mutable
  stored properties. Treat them as immutable input values and construct a new
  pack when configuration changes.
- `NoOpNetworkLogger` and the redirect/diagnostic helper constants used across
  package products are no longer public. Generated-client SPI executables now
  inherit logger and empty-interceptor defaults when those witnesses are omitted.
- `StateReducer` and `StateReduction` are package implementation details.
  Applications should own reducer vocabulary at their feature boundary.
- Redirect defaults deny HTTPS downgrade and every cross-origin proposal that
  retains an unsafe method. Other cross-origin hops strip every caller-prepared
  original header plus built-in and configured sensitive session headers. Core
  URLSession transports additionally clear every `httpAdditionalHeaders` value
  across origins so Foundation cannot re-inject a removed session default;
  same-origin values are preserved. Signed requests deny all automatic
  redirects.
- Body-dependent authentication moves to `RequestSigner` and `RequestBody`.
- `WebSocketManager.retry(_:)` now returns an optional `WebSocketRetryResult`.
  Accepted explicit retries use a new task ID, leave the source task terminal,
  and return a bounded event stream registered before the replacement transport
  resumes. This removes the late-registration gap; the configured overflow
  policy still governs nonterminal event delivery.
- `WebSocketHandshakeRequestAdapter` is now `async throws`. Adapter failures
  enter the task's normal connection-failure/reconnect lifecycle, while a
  disconnect or shutdown that wins during suspension discards the stale
  callback result.
- The nested `InnoNetworkCodegen` package and `#endpoint` expression macro are
  removed. `@APIDefinition` now ships from the root `InnoNetwork` product,
  requires explicit `auth: .anonymous`, `.optional`, or `.required`, and
  requires the endpoint struct to declare `APIResponse`.
- Public optional overloads of
  `EndpointPathEncoding.percentEncodedSegment(_:)` are removed. Callers must
  decide the nil behavior before path construction.
- The raw-string `NetworkClient.request(_:method:tag:)` overload is removed.
  Named API catalog entries use explicit macro-assisted `APIDefinition`
  structs; one-off or runtime-composed requests use `EndpointBuilder` with an
  explicit authentication choice.
- `NetworkClient` is request-only. Multipart execution uses the independent
  `UploadNetworkClient` capability. `DefaultNetworkClient` and
  `StubNetworkClient` support both, while request-only conformers no longer
  implement unrelated upload requirements.
- The raw `ConcurrencyTokenBucket` semaphore is package-owned. Construct
  `ConcurrencyLimitExecutionPolicy(maxConcurrent:)` directly and register it
  in `ResiliencePack.customExecutionPolicies`. Reusing one policy value shares
  its concurrency cap across clients.
- `NetworkConfiguration.responseBodyLimit` is removed. The
  `responseBodyBufferingPolicy` enum is the single source of truth for both
  collection mode and its optional byte ceiling.
- The WebSocket manager's no-op
  `handleBackgroundSessionCompletion(_:completion:)` method is removed.
  Foundation background-transfer callbacks remain owned by `DownloadManager`.
- The unused `WebSocketConfiguration.sessionIdentifier` property and matching
  advanced builder field are removed. WebSocket sessions remain foreground
  sessions managed internally.
- The duplicate `DownloadConfiguration.default` and
  `WebSocketConfiguration.default` aliases are removed. Explicit
  configuration uses `safeDefaults()`; zero-argument manager initialization
  remains unchanged.
- The duplicate `DownloadManager.make(configuration:)` factory is removed.
  Construct the manager with the equivalent throwing
  `DownloadManager(configuration:)` initializer.
- `PersistentResponseCacheStatistics` is a cache-owned read-only snapshot.
  Obtain it from `await cache.statistics()` instead of constructing synthetic
  values through a public initializer.
- `CircuitBreakerOpenError` remains publicly inspectable, but its initializer
  is package-owned because only the built-in circuit breaker produces this
  wrapped diagnostic.
- The direct 21-parameter `WebSocketConfiguration` initializer is no longer
  public. Start from `safeDefaults()` or use `advanced(_:)` for explicit
  tuning.
- `WebSocketTask` construction is manager-owned. Obtain task handles from
  `connect(url:subprotocols:)` or an accepted explicit retry result.
- The direct 22-parameter `DownloadConfiguration` initializer is no longer
  public. Start from `safeDefaults(sessionIdentifier:)` or
  `advanced(sessionIdentifier:_:)`, then apply explicit opt-ins such as
  `cellularEnabled()` or `backgroundTransfersEnabled()`.
- Task-scoped Download and WebSocket observation now has one public path:
  `events(for:)`. Manual listener subscription tokens and add/remove methods
  are no longer consumer API; manager-wide callback setters remain available.
- Download and WebSocket transition tables are manager-owned. Public state
  projections retain `isTerminal`, while `nextStates` and
  `canTransition(to:)` move to package scope.
- Optional observability disables work with an empty observer collection or a
  `nil` metrics reporter. The corresponding no-op helper implementations are
  removed.

## Macro-first, explicit endpoint contracts

- The default-enabled `Macros` package trait exposes
  `@APIDefinition(method:path:auth:)` through `import InnoNetwork`.
- The annotated struct remains the source of truth. The macro derives only
  `APIDefinition` conformance, method, percent-encoded path, session
  authentication, and supported simple payload witnesses.
- A GET/HEAD `query` or POST/PUT/PATCH/DELETE `body` stored property derives
  `Parameter` and `parameters`. OPTIONS, CONNECT, TRACE, custom, and dynamic
  methods require a complete manual `Parameter` + `parameters` pair whenever
  they carry a payload; the macro does not guess their transport meaning.
- `auth:` derives `sessionAuthentication`, but the explicit endpoint struct
  remains authoritative for `APIResponse`, stored inputs, and any custom
  transport, headers, interceptors, decoder, or policy.
- Standard method and authentication cases may use contextual (`.get`),
  type-qualified (`HTTPMethod.get`), or module-qualified
  (`InnoNetwork.HTTPMethod.get`) syntax. The macro canonicalizes recognized
  members and continues to reject arbitrary aliases during payload inference.
- Compile-time diagnostics fail closed on missing response/auth declarations,
  unsafe paths, invalid stored-property shapes, and generated-witness
  conflicts. Safe interpolation rewrites receive a Fix-It. Path properties
  whose typealiases resolve to Optional receive a targeted unwrap-and-define-
  nil-behavior error rather than a generic path-encoder constraint failure.
- Passing an endpoint struct that omitted `@APIDefinition` to either
  `NetworkClient.request` overload produces an actionable error requesting the
  macro or a manual conformance at that first provable endpoint boundary.
- Core-only consumers can request `traits: []` to exclude the macro API and
  compiler plug-in compilation. SwiftPM can still resolve or fetch
  manifest-level `swift-syntax`; traits are unified per package when another
  dependency enables them.

## Request correctness and security

- A shared URL-admission gate now covers core requests, downloads, WebSockets,
  and the OpenAPI client transport. HTTPS and WSS are the defaults; plain HTTP
  or WS requires the narrowly scoped `allowsInsecureHTTP` or
  `allowsInsecureWebSocket` opt-in on the matching configuration. Userinfo,
  fragments, ambiguous hosts, absolute-origin overrides, and raw or recursively
  encoded dot-segment traversal are rejected before transport.
- Persistent-cache `index.json` reads stop at a 16 MiB ceiling before JSON
  decoding. Oversized or structurally corrupt indexes cold-reset only the
  cache-owned index and bodies, while transient access failures preserve them.
- HTTP method tokens remain case-sensitive through retry, redirect, cache,
  coalescing, response framing, and curl diagnostics. Core streaming/buffered
  requests and the OpenAPI URLSession transport fail before I/O when Foundation
  would rewrite a differently cased token rather than transmit it faithfully.
- Every default-session WebSocket handshake redirect is re-admitted before it
  is followed. WSS-to-WS downgrade is rejected even with the plain-WS opt-in,
  cross-origin redirects strip built-in credentials and every caller-prepared
  header while preserving CFNetwork's required handshake and subprotocol
  negotiation fields, and a rejected target fails terminally without
  scheduling reconnect. Multi-hop redirects remain bound to the original
  handshake origin for credential decisions.
- `NetworkConfiguration.safeDefaults(baseURL:)` and
  `NetworkConfiguration.advanced(...)` collect responses,
  including file-upload responses, through a streaming path capped at 5 MiB by
  default. Callers that intentionally accept unbounded responses must opt out
  explicitly with `.streaming(maxBytes: nil)` or `.buffered(maxBytes: nil)`.
  The former zero-argument spellings no longer compile in 5.0. For inline
  requests, `.buffered(maxBytes:)` validates only after `data(for:)` has
  buffered the body; bounded streaming validates while receiving and cancels
  the underlying task when the ceiling is crossed.
- A bounded file upload uses a streamed data task with the file supplied as an
  `httpBodyStream` and an explicit `Content-Length`, allowing an oversized
  response to cancel early. An explicitly unbounded file upload keeps the
  native `upload(for:fromFile:)` path. External custom `URLSessionProtocol`
  implementations fail closed for bounded file uploads.
- `MockURLSession` and VCR replay mode from `InnoNetworkTestSupport` remain
  usable with the bounded `safeDefaults` profile. Because their deterministic
  fixture data is already resident in memory, the executor uses their buffered
  path and applies the configured ceiling before cache insertion, interceptors,
  or decoding. VCR record mode forwards to a live backing session and therefore
  fails closed under bounded streaming; use an explicitly reviewed
  `.buffered(maxBytes:)` recording profile. Other custom sessions still need
  streaming support for a bounded streaming policy.
- GET, HEAD, and TRACE requests with an encoded body now fail before transport.
  GET and HEAD default to query transport; all other methods require an
  explicit payload decision when that default is not correct.
- `DownloadConfiguration.safeDefaults` and `advanced` now select a secure
  foreground session. Foreground redirect hops pass through
  `DefaultRedirectPolicy` and URL admission; rejected downgrade, unsafe replay,
  or traversal targets surface as `DownloadError.invalidURL`, cancel the task,
  and do not retry or stage a 3xx response body as a successful file.
- `backgroundTransfersEnabled()` is the single public continuation/security
  opt-in. Foundation follows background redirects automatically without
  calling the redirect delegate, so per-hop preflight cannot be enforced.
  Initial and final URLs remain validated where applicable, but that does not
  retroactively protect an intermediate redirect hop already contacted by the
  system.
- App Group reattachment is single-owner and OS-driven; proactive transfer
  handoff is not supported. Alternate app/extension owners must use the same
  App Group-backed `persistenceBaseDirectoryURL` as well as the same Foundation
  `sharedContainerIdentifier`, so restored system tasks retain their matching
  InnoNetwork logical records.
- `DownloadTask` instances are manager-owned handles. Their initializer is no
  longer public; create downloads through `DownloadManager.download(...)` so
  persistence, callback correlation, and URLSession ownership are established
  together.
- The encoded request body is finalized before signing. Configuration and
  endpoint interceptors run first, refresh-token state is applied next, and
  configuration signers then endpoint signers observe the exact body the
  transport will send.
- Data and stable file-backed signing are supported. Caller-owned files are
  copied to immutable, execution-scoped snapshots and cleaned up after the
  attempt. Opaque `httpBodyStream` values are rejected.
- HMAC, request-minted JWT, and AWS SigV4 reference signers use the new late
  signing contract. AWS SigV4 supports empty, data, and file bodies, including
  S3 payload hashing.
- Signed traffic bypasses InnoNetwork response-cache reads and writes,
  in-flight coalescing, and URLSession cache storage. It adds `no-store` and
  rejects automatic redirect follow-ups that would bypass the async signer.
- `APIDefinition.transport` is captured once per executable, so encoding,
  content-type selection, decoding, and retries share one policy value.

## OpenAPI and HTTP semantics

- `OpenAPIRestOperation` now requires an explicit
  `sessionAuthentication: SessionAuthentication` witness. The bundled preview
  generator emits `.anonymous` because it does not interpret OpenAPI security
  schemes; generated operations must be reviewed and changed to `.optional` or
  `.required` where the service contract demands it.
- `InnoNetworkClientTransport` returns no `HTTPBody` for HEAD responses,
  successful CONNECT `2xx`, informational `1xx`, `204`, `205`, and `304`, even
  when a server sends bytes or a misleading `Content-Length`.
- `InnoNetworkClientTransport` validates its base URL and relative request
  target with the same HTTPS/origin/traversal policy as the core client. Its
  explicit `allowsInsecureHTTP` initializer option is intended only for scoped
  local, LAN, or staging use.
- Generated-client redirects now apply `DefaultRedirectPolicy` and repeat URL
  admission on every hop. Cross-origin hops strip original request headers and
  clear values configured through `URLSessionConfiguration.httpAdditionalHeaders`.
  Policy or admission failures surface as the redacted `redirectRejected`
  error; background URLSession instances fail before dispatch with
  `backgroundSessionUnsupported` because Foundation does not expose their
  redirect hops to the task delegate.
- `swift-http-types` remains a direct compatible 1.x dependency of the optional
  `InnoNetworkOpenAPI` product. Core request, header, response, and endpoint
  models do not expose HTTPTypes.

## Privacy, cache, and owned storage

- `ResponseCacheKey` preserves query-item order. Targets such as
  `?a=1&b=2` and `?b=2&a=1` no longer share a cache slot. Persistent cache
  format version 4 HMAC-protects the complete raw query on disk and cold-resets
  version-3-or-older indexes so legacy raw query material is not retained.
- Diagnostic curl output omits request bodies and redacts query values and
  sensitive headers by default. `includesBody` and `includesQueryValues` are
  explicit debugging opt-ins; URL userinfo and fragments are always removed.
- Observability events redact query values, userinfo, fragments, and JWT-like
  path tokens. Failure events expose stable categories instead of decoder,
  trust, URL, or response-derived payload text.
- Persistent-cache-owned indexes, body/key files, and subdirectories, plus
  download-owned metadata and staging directories, are excluded from backup on
  Darwin. On iOS, tvOS, watchOS, and visionOS they also receive the
  `.completeUntilFirstUserAuthentication` Data Protection class. macOS applies
  backup exclusion but not an iOS-family file-protection class. A
  caller-supplied cache root is not backup-excluded because it may be shared
  with app files; final downloaded payload metadata is not rewritten.

## Lifecycle and concurrency hardening

- Download persistence retains an open session-directory descriptor and uses
  descriptor-relative, no-follow operations for every authoritative metadata
  read, write, lock, rename, unlink, and size check. Owned root/session/file
  symlinks, hard links, and blocking FIFO entries are rejected, atomic temporary and quarantine moves verify inode
  identity, and replacing the visible parent cannot redirect I/O to an App
  Group sibling. The caller-provided base path is canonicalized once as the
  explicit trusted anchor. This is a single, cooperating-owner boundary; a
  concurrently malicious process with the same App Group privileges requires
  isolation in a private container or behind an exclusive IPC persistence broker.
- Download persistence preserves checkpoint and append-log files when startup
  encounters Data Protection, permission, lock, or transient I/O failures.
  Corrupt-log recovery commits the valid prefix as a durable checkpoint before
  quarantining or resetting the source log.
- Download completion files are staged before delegate completion returns;
  pause/resume and temporary-file cleanup are bounded and cancellation-safe.
- Download restoration is cancellation-aware, and opaque resume/system tasks
  are accepted only when their original and current requests match the
  persisted admitted URL. Shutdown waits for already-admitted mutations,
  removes their persistence records, and cannot be followed by a late task
  resume from a suspended persistence call.
- Background restore keeps missing-system failures provisional until
  `urlSessionDidFinishEvents(forBackgroundURLSession:)`, the Foundation boundary
  for events queued during session reattachment. UIKit completion handlers are
  registered synchronously and are never satisfied by an unscoped finish from
  an earlier batch.
- Download success uses an attempt-scoped deterministic journal and SHA-256
  commit metadata. Live and restored `.finished` receipts survive until terminal
  delivery admission and are removed only by an exact metadata/outcome
  acknowledgment. Destination-integrity failures preserve recovery evidence
  and surface as `DownloadError.fileSystemError`; policy-rejected journals are
  explicitly abandoned and cleaned so retry remains available.
- Shared containers do not permit simultaneous managers for one background
  session identifier. Exactly one process owns an identifier at a time, and
  concurrent logical tasks require distinct final destination paths.
- Download persistence and completion staging keep conventional lowercase
  reverse-DNS session identifiers unchanged, but map path-like, uppercase,
  oversized, empty, or non-ASCII identifiers to one deterministic SHA-256
  directory component. An identifier therefore cannot create nested storage,
  escape the configured root, or alias another session by case.
- Download delegate progress is coalesced per task segment while completions
  remain lossless and FIFO. Final completed, failed, and cancelled events are
  admitted and atomically seal the partition, so a later progress callback
  cannot evict the terminal outcome from bounded listeners or streams.
- App-facing download callbacks run on per-task ordered delivery lanes. Slow
  progress and terminal handlers no longer block the URLSession delegate FIFO
  or the system background-session completion handler. Pre-transport waiting
  and downloading handlers retain their admission semantics, and external
  shutdown still drains all accepted callbacks.
- Download manager callbacks may initiate shutdown without self-joining the
  restoration or delegate consumer task. External shutdown callers still wait
  for callback drain, URLSession invalidation, and the complete teardown
  boundary.
- WebSocket manual disconnect and manager shutdown have bounded teardown and
  deliver one final terminal shutdown error to every consumer present in the
  publication snapshot, even when `.dropNewest` queues are saturated.
- Automatic WebSocket reconnect keeps the same public task and listeners while
  rotating the transport generation; explicit retry creates a new logical task
  and is accepted only once by the source task's owning manager.
- WebSocket terminal cleanup publishes and enqueues the final outcome, closes
  the task partition, removes registry state, releases the lifecycle gate, and
  only then invokes snapshotted manager callbacks. External shutdown waits for
  those callbacks, while listener and `AsyncStream` delivery remains async.
- Reconnect-budget exhaustion publishes one authoritative
  `.error(.maxReconnectAttemptsExceeded)` or
  `.error(.reconnectWindowExceeded)` without a preceding synthetic
  `.disconnected` event.
- Pong event publication is attempted before the matching snapshotted manager
  handler runs. Ordinary event overflow still applies, so no listener-versus-
  handler execution order or listener delivery guarantee is implied.
- Refresh-token coordination uses explicit generations so late callers reuse a
  completed refresh cohort without issuing another refresh.
- Persistent-cache key, index, and open-time body access failures caused by
  Data Protection, permissions, or transient I/O preserve existing state and
  fail initialization. A transient live body read returns a miss without
  scrubbing the entry, so a later retry can succeed. Index symlinks, directories,
  FIFOs, and other non-regular entries cold-reset owned state without following
  or blocking on them. The pre-transport cache
  lookup result is reused instead of reading storage twice.
- Circuit-breaker half-open trials preserve configured 4xx hysteresis.
- Request event partition retirement preserves terminal events already queued
  behind a slow observer. Finishing a request waits for partition-to-observer
  handoff, keeps the closed lifecycle installed across teardown reentrancy,
  and does not wait for observer handler execution.
- Cancellation, refresh coalescing, and WebSocket tests use continuation gates
  instead of scheduler timing assumptions.

## Release engineering

- `docs/releases/5.0.0.md` remains in draft status until the release is
  deliberately approved. Release validation accepts only the exact ready
  status marker at the top of the file, so the current draft intentionally
  fails the release workflow before its build and GitHub Release publication.
  Repository policy must still prevent pushing a tag from a draft commit. A
  marker-only edit also fails: the validator reads the tagged Git tree and
  requires README, API stability, CHANGELOG, security support, symbol-baseline,
  migration, status-line, and release-date claims to form one coherent ready
  state.
- Release tags must be unprefixed annotated SemVer tags. The tagged commit must
  exactly match the freshly fetched `origin/main` HEAD and contain matching
  versioned release notes.
- Default-trait and core-only dependency profiles produce deterministic
  CycloneDX 1.5 SBOMs without local paths. Both artifacts are signed and
  attached to releases.
- Coverage fails closed on missing, empty, or absolute-path LCOV output.
  Benchmarks build in release mode, and all guarded measurements are checked.
- `bash Scripts/run_local_release_preflight.sh --full` reproduces the pre-tag
  release contracts, coverage, 10% guarded benchmarks, two SBOM profiles,
  eight public DocC archives, and five-platform builds in one local command.
  Tag identity, signing, and publication remain GitHub-only gates.
- `Scripts/build_consumer_examples.sh` discovers and builds every independent
  example package for CI, release validation, and local preflight. Adding an
  example no longer requires copying its path into multiple workflow steps.
- `Examples/MacroAdopterSmoke` runs an independent public-product consumer at
  CI, release, and local-preflight time, proving that explicit
  `@APIDefinition` endpoint structs execute GET/query and POST/body/auth flows
  through `DefaultNetworkClient` without package-only access.
- `Benchmarks/guarded-benchmarks.txt` defines the protected performance set.
  `Scripts/run_with_guarded_benchmarks.py` injects the ordered set into PR,
  scheduled/manual, release, contributor-documentation, and local-preflight
  commands. A fail-closed contract rejects direct declarations that bypass the
  runner as well as guard identifiers absent from the default baseline.
- Coverage uploads use GitHub OIDC in dedicated artifact-only jobs with a
  checksum-pinned Codecov CLI, and the dead-code gate installs a
  checksum-pinned official Periphery release.
- Apple platform smoke builds require macOS, iOS, tvOS, watchOS, and visionOS
  SDKs on the pinned runner. tvOS, watchOS, and visionOS cross-compile every
  public library product at the package deployment floor, independent of
  hosted-runner simulator runtime retention. Missing SDKs and compile failures
  fail both CI and release validation rather than becoming advisory skips.
- A fail-closed Apple platform build contract derives those floors from
  `Package.swift` and rejects matrix, local-preflight, or cross-build-helper
  destinations and target triples that drift from the package declaration.
- Every SwiftPM example manifest now declares the package floors: iOS 16,
  macOS 14, tvOS 16, watchOS 9, and visionOS 1. The floor gate also requires
  CI and release validation to build every discovered example, while release
  validation builds and tests the bundled `openapi-to-innonetwork` CLI.
- The root `Package.resolved` is tracked for reproducible CI and as GitHub's
  Swift dependency-graph manifest. CI and release validation reject lock drift,
  deletion, and untracked recreation. Dependency review is a blocking pull-request gate across runtime,
  development, and unknown scopes; a rejected dependency or unavailable graph
  fails the job.
- DocC builds and publishes all eight public products:
  `InnoNetwork`, `InnoNetworkAuthAWS`, `InnoNetworkDownload`,
  `InnoNetworkWebSocket`, `InnoNetworkPersistentCache`,
  `InnoNetworkOpenAPI`, `InnoNetworkTrust`, and
  `InnoNetworkTestSupport`. The ordered product contract is shared by local,
  release, and Pages gates; it must match SwiftPM library products, and every
  named product must produce exactly one archive.
- Public and SPI symbol ledgers are checked independently. Built-in generated
  executable adapters that do not need to cross a package boundary are
  package-scoped rather than exposed as generated-client SPI.

## Validation contract

The release gate runs the root suite in serial coverage mode and across four
bounded target shards, plus the root macro suite and negative compile fixtures,
default trait graph/core-only build checks, public API/doc synchronization,
stable example smokes, release-mode benchmarks, all-product DocC builds,
release-tag fixtures, and default/core-only SBOM fixtures. Live network tests remain
isolated in explicit scheduled or manually dispatched smoke lanes.

## Upgrade checklist

1. Apply every source migration in
   [`Migration-5.0.0.md`](../Migration-5.0.0.md).
2. Classify every endpoint as `.anonymous`, `.optional`, or `.required`, and
   review generated OpenAPI operations instead of accepting `.anonymous`
   mechanically.
3. Audit exhaustive `HTTPMethod` switches, HEAD/custom-method payloads, and
   any intentional HTTP/WS development endpoints.
4. Exercise signed uploads across retries and refresh replays.
5. Verify endpoints that depend on redirects.
6. Run the app's integration suite with response cache and request coalescing
   enabled.
7. Confirm that responses larger than 5 MiB have an intentional bounded or
   explicit-unbounded policy.
8. Review the public stability ledger in [`API_STABILITY.md`](../../API_STABILITY.md)
   before choosing `.upToNextMinor` or `.upToNextMajor` pinning.
