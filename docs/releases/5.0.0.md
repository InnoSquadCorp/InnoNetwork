# InnoNetwork 5.0.0 Release Notes

InnoNetwork 5.0 is a deliberate breaking release that makes wire-request
identity and release integrity explicit. It keeps the Apple platform floors at
iOS 16, macOS 14, tvOS 16, watchOS 9, and visionOS 1, and requires Swift 6.2.

Read [`docs/Migration-5.0.0.md`](../Migration-5.0.0.md) before upgrading from
4.x.

## Breaking changes

- `RequestExecutionNext.execute(_:)` is replaced by `execute()`. Execution
  policies can observe, short-circuit, or replay the request captured by the
  executor; request mutation belongs in `RequestInterceptor`.
- The seven deprecated `NetworkConfiguration.with(...)` methods are removed.
  Use named configuration packs with `NetworkConfiguration.advanced(...)`.
- `StateReducer` and `StateReduction` are package implementation details.
  Applications should own reducer vocabulary at their feature boundary.
- Redirect defaults deny HTTPS downgrade and unsafe cross-origin `307`/`308`
  replay. Signed requests deny all automatic redirects.
- Body-dependent authentication moves to `RequestSigner` and `RequestBody`.
- `WebSocketManager.retry(_:)` now returns an optional `WebSocketRetryResult`.
  Accepted explicit retries use a new task ID, leave the source task terminal,
  and return a bounded event stream registered before the replacement transport
  resumes. This removes the late-registration gap; the configured overflow
  policy still governs nonterminal event delivery.

## Request correctness and security

- The encoded request body is finalized before signing. Configuration and
  endpoint interceptors run first, refresh-token state is applied next, and
  configuration signers then endpoint signers observe the exact body the
  transport will send.
- Data and stable file-backed signing are supported. Caller-owned files are
  copied to immutable, execution-scoped snapshots and cleaned up after the
  attempt. Opaque `httpBodyStream` values are rejected.
- HMAC, request-minted JWT, and AWS SigV4 reference signers use the new late
  signing contract. AWS SigV4 supports empty, data, and file bodies, including
  S3 payload hashing.
- Signed traffic bypasses InnoNetwork response-cache reads and writes,
  in-flight coalescing, and URLSession cache storage. It adds `no-store` and
  rejects automatic redirect follow-ups that would bypass the async signer.
- `APIDefinition.transport` is captured once per executable, so encoding,
  content-type selection, decoding, and retries share one policy value.

## Lifecycle and concurrency hardening

- Download completion files are staged before delegate completion returns;
  pause/resume and temporary-file cleanup are bounded and cancellation-safe.
- WebSocket manual disconnect and manager shutdown have bounded teardown and
  deliver one final terminal shutdown error to every consumer present in the
  publication snapshot, even when `.dropNewest` queues are saturated.
- Automatic WebSocket reconnect keeps the same public task and listeners while
  rotating the transport generation; explicit retry creates a new logical task
  and is accepted only once by the source task's owning manager.
- WebSocket terminal cleanup publishes and enqueues the final outcome, closes
  the task partition, removes registry state, releases the lifecycle gate, and
  only then invokes snapshotted manager callbacks. External shutdown waits for
  those callbacks, while listener and `AsyncStream` delivery remains async.
- Reconnect-budget exhaustion publishes one authoritative
  `.error(.maxReconnectAttemptsExceeded)` or
  `.error(.reconnectWindowExceeded)` without a preceding synthetic
  `.disconnected` event.
- Pong event publication is attempted before the matching snapshotted manager
  handler runs. Ordinary event overflow still applies, so no listener-versus-
  handler execution order or listener delivery guarantee is implied.
- Refresh-token coordination uses explicit generations so late callers reuse a
  completed refresh cohort without issuing another refresh.
- Cache key-read failures preserve existing state, and the pre-transport cache
  lookup result is reused instead of reading storage twice.
- Circuit-breaker half-open trials preserve configured 4xx hysteresis.
- Request event partition retirement preserves terminal events already queued
  behind a slow observer. Finishing a request waits for partition-to-observer
  handoff, keeps the closed lifecycle installed across teardown reentrancy,
  and does not wait for observer handler execution.
- Cancellation, refresh coalescing, and WebSocket tests use continuation gates
  instead of scheduler timing assumptions.

## Release engineering

- Release tags must be unprefixed annotated SemVer tags reachable from the
  fetched `origin/main`, and the tagged commit must contain matching versioned
  release notes.
- Root and codegen dependency graphs produce deterministic CycloneDX 1.5 SBOMs
  without local paths. Both artifacts are signed and attached to releases.
- Coverage fails closed on missing, empty, or absolute-path LCOV output.
  Benchmarks build in release mode, and all guarded measurements are checked.
- Coverage uploads use GitHub OIDC in dedicated artifact-only jobs, and the
  dead-code gate installs a checksum-pinned official Periphery release.
- Apple platform smoke builds require macOS and iOS on the pinned runner and
  build tvOS, watchOS, and visionOS whenever their hosted Xcode components are
  installed; an unavailable optional component is reported explicitly.
- DocC builds and publishes all eight public products:
  `InnoNetwork`, `InnoNetworkAuthAWS`, `InnoNetworkDownload`,
  `InnoNetworkWebSocket`, `InnoNetworkPersistentCache`,
  `InnoNetworkOpenAPI`, `InnoNetworkTrust`, and
  `InnoNetworkTestSupport`.
- `Packages/InnoNetworkCodegen` remains an experimental nested package for
  complete local checkouts; the root release tag does not vend its product.

## Validation contract

The release gate runs the root suite in serial coverage and four-worker
parallel modes, plus the codegen suite, public API/doc synchronization, stable
example smokes, release-mode benchmarks, all-product DocC builds, release-tag
fixtures, and root/codegen SBOM fixtures. Live network tests remain isolated in
explicit scheduled or manually dispatched smoke lanes.

## Upgrade checklist

1. Apply every source migration in
   [`Migration-5.0.0.md`](../Migration-5.0.0.md).
2. Exercise signed uploads across retries and refresh replays.
3. Verify endpoints that depend on redirects.
4. Run the app's integration suite with response cache and request coalescing
   enabled.
5. Review the public stability ledger in [`API_STABILITY.md`](../../API_STABILITY.md)
   before choosing `.upToNextMinor` or `.upToNextMajor` pinning.
