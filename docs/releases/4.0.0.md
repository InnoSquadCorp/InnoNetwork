# InnoNetwork 4.0.0 Release Notes

Status: Unreleased upcoming public release.

`4.0.0` is the next public release line for InnoNetwork. The tag has not been
published yet; the latest published public release remains `3.0.1`. Until the
tag is cut, consumers should pin `release/v4.0` or a repository revision.

## Highlights

- **Swift 6 language mode** is enabled for every package target.
- **`safeDefaults`** is the recommended configuration entry point for core,
  download, and WebSocket integrations.
- **Stable request execution** stays centered on `NetworkClient.request(_:)`
  and `NetworkClient.upload(_:)`.
- **Per-endpoint `acceptableStatusCodes`** lets endpoints treat codes such as
  `304 Not Modified` as success without changing the session-wide default.
- **`WebSocketCloseCode`** is part of the public WebSocket contract, including
  the full RFC 6455 range and application-defined `.custom(UInt16)` codes.
- **WebSocket heartbeat observability** is stable through `.ping`, `.pong`,
  and `.error(.pingTimeout)` events.
- **WebSocket reconnect counters** split into `attemptedReconnectCount` and
  `successfulReconnectCount`. `reconnectCount` remains as a deprecated
  compatibility alias.
- **WebSocket bounded send queues** cap per-task in-flight send concurrency
  with `.fail` or `.dropNewest` overflow behavior.
- **Public-key pinning host matching** now has an explicit strategy:
  `.unionAllMatches` preserves existing exact + parent-domain union behavior,
  while `.mostSpecificHost` isolates the exact host or longest matching parent.
- **`DownloadManager` is a public actor** and `DownloadManager.shared` no
  longer crashes on duplicate session identifiers; use
  `DownloadManager.make(configuration:)` when explicit failure handling is
  needed.
- **Multipart and streaming readiness** includes `MultipartUploadStrategy`,
  `StreamingResumePolicy`, failure-payload redaction by default, and explicit
  persistence fsync policy.
- **DocC and operational docs** cover retry decisions, error classification,
  trust policies, background downloads, persistence, WebSocket close codes,
  reconnect, platform support, query encoding, and production rollout.
- **Release artifacts** are prepared with CycloneDX SBOM generation and
  sigstore cosign keyless signing.

## Migration Notes

The 4.0.0 line includes source-breaking cleanup before the first 4.x tag:

| Pre-4.0 behavior | 4.0.0 behavior | Recovery |
|------------------|----------------|----------|
| WebSocket disconnect APIs accepted `URLSessionWebSocketTask.CloseCode`. | They accept `WebSocketCloseCode`. | Rebuild first; shared case names usually keep call sites source-compatible. |
| `WebSocketTask.closeCode` returned Apple's close-code enum. | It returns `WebSocketCloseCode?`. | Switch on the new enum and handle `.serviceRestart`, `.tryAgainLater`, and `.custom(_)` where relevant. |
| `WebSocketManager.receive(_:)` could call the URL task directly. | The public direct receive API is removed. | Observe frames through `events(for:)`, `addEventListener(for:listener:)`, or message/string callbacks. |
| `NetworkError` response payloads remained attached by default. | Failure payloads are redacted by default. | Set `NetworkConfiguration.captureFailurePayload = true` for diagnostic builds that need bodies. |
| `DownloadManager` was a `final class : NSObject, Sendable`. | It is a `public actor`. | Public methods were already `async`; direct subclassing/KVO-style designs need a different owner. |
| `WebSocketTask.reconnectCount` was the sole counter. | It is a deprecated alias of `attemptedReconnectCount`. | Use `attemptedReconnectCount` for per-cycle attempts or `successfulReconnectCount` for lifetime successes. |

Use [MIGRATION_v4.md](../../MIGRATION_v4.md) for the full call-site guide and
[docs/MIGRATION_POLICY.md](../MIGRATION_POLICY.md) for migration-document
expectations.

## Stability Contract

The stable public API surface is defined in
[API_STABILITY.md](../../API_STABILITY.md). Lower-level request execution hooks
and generated-client SPI may exist in the source tree while the package is
being prepared, but they are not part of the default `4.0.0` stable contract.

Swift OpenAPI Generator integration is official as a recipe, not a separate
4.0.0 product: wrap operations as `APIDefinition` where possible, and treat the
SPI execution hook as revision-pinned generated-client support outside the
stable import contract.

## Deferred

- WebSocket `permessage-deflate` (RFC 7692) is deferred because
  `URLSessionWebSocketTask` does not expose deflate negotiation. The v5
  candidate is an optional `InnoNetworkWebSocketNIO` product so the existing
  URLSession-based WebSocket product stays stable.
- A dedicated OpenAPI adapter package/product remains a 4.x+ roadmap
  candidate after the 4.0.0 contract is tagged.
