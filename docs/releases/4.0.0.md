# InnoNetwork 4.0.0 Release Notes

InnoNetwork 4.0.0 is the first public release. The package targets Apple
platforms only and is built around Swift Concurrency, explicit transport
policies, and operational visibility from prototype to production.

## Highlights

- **Swift 6 language mode** is enabled for every package target.
- **`safeDefaults`** is the recommended configuration entry point for core,
  download, and WebSocket integrations.
- **Single transport entry point** on `APIDefinition`. Endpoints override
  `transport: TransportPolicy<APIResponse>` instead of six separate
  properties (`contentType`, `requestEncoder`, `queryEncoder`,
  `queryRootKey`, `decoder`, `responseDecoder`); the default is method-aware
  (`GET` → `.query()`, otherwise `.json()`).
- **Tag-based cancellation** through `CancellationTag`,
  `request(_:tag:)`, `upload(_:tag:)`, and `cancelAll(matching:)` so a
  screen, feature, or user session can drop just its own subset of
  in-flight requests.
- **Streaming pipeline extracted** into `StreamingExecutor`. Same observable
  behaviour as before but unit-testable on its own.
- **Stable request execution** is centered on `NetworkClient.request(_:)`
  and `NetworkClient.upload(_:)`.
- **Built-in resilience policies** add opt-in auth refresh, request
  coalescing, response caching, and circuit breaking without exposing a public
  generic execution-policy protocol.
- **Auth refresh** through `RefreshTokenPolicy` applies a current token,
  coordinates a single refresh across concurrent `401` responses, and replays
  each request at most once.
- **Request coalescing** through `RequestCoalescingPolicy` can fan out one raw
  transport result to identical in-flight requests. Authorization remains part
  of the deduplication key by default.
- **Response cache** through `ResponseCachePolicy`, `ResponseCache`,
  `CachedResponse`, `ResponseCacheKey`, and `InMemoryResponseCache` supports
  opt-in GET caching, ETag revalidation, `304` body substitution,
  stale-while-revalidate, locale separation, fragment stripping, and
  fingerprinted authorization keys. The pipeline also honours the response
  `Vary` header (RFC 9111 §4.1): `Vary: *` responses are not stored, and
  concrete `Vary` headers capture a request-header snapshot that future
  lookups must match. GET responses with cacheable whole-response status codes
  can be stored, `Cache-Control: no-store` invalidates existing entries, and
  `Cache-Control: no-cache` forces revalidation before reuse.
- **Circuit breaker** through `CircuitBreakerPolicy` short-circuits repeated
  per-host transport/server failures. Open circuits surface through
  `NetworkError.underlying` wrapping `CircuitBreakerOpenError`.
- **Multipart response decoding** through `MultipartResponseDecoder` provides a
  buffered `multipart/*` parser for response bodies.
- **Optional macros** through the separate `Packages/InnoNetworkCodegen` package
  expose `@APIDefinition` and `#endpoint` while keeping `swift-syntax` out of
  the root runtime package graph.
- **Request construction contracts** now fix path joining, query handling,
  `Content-Type` attachment, and header precedence for signing-sensitive
  integrations.
- **Path encoding hardening** through the new public
  `EndpointPathEncoding.percentEncodedSegment(_:)` helper and the package
  `EndpointPathBuilder.makeURL` composer. Path literals containing raw
  whitespace or non-ASCII scalars are now percent-encoded automatically
  instead of failing `URLComponents.url` resolution and surfacing as
  `NetworkError.invalidRequestConfiguration`. Existing percent-escape
  sequences are preserved, and literals containing `?` or `#` still throw
  so query/fragment composition continues to flow through the dedicated
  encoders. Dynamic placeholder values supplied via `@APIDefinition` are
  routed through the segment helper so slashes inside an identifier no
  longer change the path hierarchy.
- **Retry idempotency** defaults to safe methods plus unsafe methods with an
  `Idempotency-Key`; method-agnostic retry is explicit opt-in.
- **Per-endpoint `acceptableStatusCodes`** lets endpoints treat codes such as
  `304 Not Modified` as success without changing the session-wide default.
- **`WebSocketCloseCode`** is part of the public WebSocket contract, including
  the full RFC 6455 range and application-defined `.custom(UInt16)` codes.
- **WebSocket heartbeat observability** is stable through `.ping`, `.pong`,
  and `.error(.pingTimeout)` events.
- **WebSocket reconnect counters** split into `attemptedReconnectCount` and
  `successfulReconnectCount`.
- **WebSocket bounded send queues** cap per-task in-flight send concurrency
  with `.fail` or `.dropNewest` overflow behavior.
- **Public-key pinning host matching** has an explicit strategy:
  `.unionAllMatches` covers exact and parent-domain pins as a union, while
  `.mostSpecificHost` isolates the exact host or longest matching parent.
- **`DownloadManager` is a public actor**; `DownloadManager.shared` falls back
  to a process-unique identifier on duplicate session-identifier conflicts so
  the singleton stays usable. Use `DownloadManager.make(configuration:)` when
  explicit failure handling is needed.
- **Multipart and streaming readiness** includes `MultipartUploadStrategy`,
  `StreamingResumePolicy`, failure-payload redaction by default, and explicit
  persistence fsync policy.
- **DocC and operational docs** cover retry decisions, error classification,
  trust policies, background downloads, persistence, WebSocket close codes,
  reconnect, platform support, query encoding, and production rollout.
- **Release artifacts** are prepared with CycloneDX SBOM generation and
  sigstore cosign keyless signing.

## Stability Contract

The stable public API surface is defined in
[API_STABILITY.md](../../API_STABILITY.md). Lower-level request execution hooks
may exist in the source tree, but they are not part of the default `4.0.0`
stable contract.

No public generic execution-policy protocol is exposed in 4.0.0. The supported
public contract is the built-in policy surface on `NetworkConfiguration` and
`NetworkConfiguration.AdvancedBuilder`. Resilience policies default to
disabled/no-op behavior, and no new `NetworkError` enum case is added for open
circuit failures.

`InnoNetworkCodegen` is a separate compile-time package. Macro helpers require
`import InnoNetworkCodegen`; depending only on the root `InnoNetwork` package
does not resolve or build `swift-syntax`.

Swift OpenAPI Generator integration is supported as a recipe: wrap operations
as `APIDefinition` where possible, and treat SPI execution hooks as
revision-pinned generated-client support outside the stable import contract.

## Deferred

- WebSocket `permessage-deflate` (RFC 7692) is deferred because
  `URLSessionWebSocketTask` does not expose deflate negotiation. The
  candidate is an optional `InnoNetworkWebSocketNIO` product so the existing
  URLSession-based WebSocket product stays stable.
- A dedicated OpenAPI adapter package/product remains a roadmap candidate.
- Server freshness directive precedence, custom cache-key policy, observability
  exporter examples, benchmark regression automation, download storage
  operations, WebSocket protocol-surface documentation, a streaming multipart
  response decoder, and Hummingbird in-process integration tests remain
  follow-up work.
