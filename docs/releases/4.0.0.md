# InnoNetwork 4.0.0 Release Notes

Status: Unreleased upcoming public release.

`4.0.0` is the next public release line for InnoNetwork. The tag has not been
published yet, so consumers should pin a branch or revision until the release
is cut.

## Highlights

- Swift 6 language mode is enabled for every package target.
- `safeDefaults` is the recommended configuration entry point for core,
  download, and WebSocket integrations.
- `NetworkClient.request(_:)` and `NetworkClient.upload(_:)` are the stable
  request execution APIs for application call sites.
- `WebSocketCloseCode` is part of the public WebSocket contract.
- WebSocket heartbeat observability is part of the stable 4.0 contract through
  `.ping`, `.pong`, and `.error(.pingTimeout)` events.
- Protocol Buffers support is prepared as a separate package,
  `InnoNetworkProtobuf`.

## Stability Contract

The stable public API surface is defined in
[API_STABILITY.md](../../API_STABILITY.md). Lower-level request execution
hooks and richer WebSocket context payloads may exist in the source tree while
the package is being prepared, but they are not part of the `4.0.0` stable
contract.

## Migration

- Use [MIGRATION_v4.md](../../MIGRATION_v4.md) for the public migration guide.
- Use [docs/MIGRATION_POLICY.md](../MIGRATION_POLICY.md) for release-note and
  migration-document expectations.

## Validation

Release validation for `4.0.0` should include:

- `bash Scripts/check_docs_contract_sync.sh`
- `swift run InnoNetworkDocSmoke`
- `swift build`

The full test suite is tracked separately until the known WebSocket
receive-loop and heartbeat contract failures in
[issue #25](https://github.com/InnoSquadCorp/InnoNetwork/issues/25) are fixed.
