# InnoNetwork 4.0.0 Release Notes

InnoNetwork 4.0.0 is the first public release. The package targets Apple
platforms only and is built around Swift Concurrency, explicit transport
policies, and operational visibility from prototype to production.

## Highlights

- **Swift 6 language mode** is enabled for every package target.
- **`safeDefaults`** is the recommended configuration entry point for core,
  download, and WebSocket integrations.
- **Single transport entry point** on `APIDefinition`. Endpoints override
  `transport: TransportPolicy<APIResponse>` instead of six separate
  properties (`contentType`, `requestEncoder`, `queryEncoder`,
  `queryRootKey`, `decoder`, `responseDecoder`); the default is method-aware
  (`GET` → `.query()`, otherwise `.json()`).
- **Tag-based cancellation** through `CancellationTag`,
  `request(_:tag:)`, `upload(_:tag:)`, and `cancelAll(matching:)` so a
  screen, feature, or user session can drop just its own subset of
  in-flight requests.
- **Streaming pipeline extracted** into `StreamingExecutor`. Same observable
  behaviour as before but unit-testable on its own.
- **Stable request execution** is centered on `NetworkClient.request(_:)`
  and `NetworkClient.upload(_:)`.
- **Built-in resilience policies** add opt-in auth refresh, request
  coalescing, response caching, and circuit breaking without exposing a public
  generic execution-policy protocol.
- **Auth refresh** through `RefreshTokenPolicy` applies a current token,
  coordinates a single refresh across concurrent `401` responses, and replays
  each request at most once. The refresh-aware coalescer lane segregates
  callers entering during a refresh-in-flight window so a peer's pre-refresh
  transport result cannot leak into another caller's resumed continuation.
- **Request coalescing** through `RequestCoalescingPolicy` can fan out one raw
  transport result to identical in-flight requests. Authorization remains part
  of the deduplication key by default.
- **Response cache** through `ResponseCachePolicy`, `ResponseCache`,
  `CachedResponse`, `ResponseCacheKey`, and `InMemoryResponseCache` supports
  opt-in GET caching, ETag revalidation, `304` body substitution,
  stale-while-revalidate, locale separation, fragment stripping, and
  fingerprinted authorization keys. The pipeline also honours the response
  `Vary` header (RFC 9111 §4.1): `Vary: *` responses are not stored, and
  concrete `Vary` headers capture a request-header snapshot that future
  lookups must match. GET responses with cacheable whole-response status codes
  can be stored, `Cache-Control: no-store` invalidates existing entries, and
  `Cache-Control: no-cache` forces revalidation before reuse. A 304
  conditional revalidation that succeeds but carries a different `Vary`
  header preserves the stored entry's headers, body, and vary snapshot —
  the executor refreshes only `storedAt`.
- **Circuit breaker** through `CircuitBreakerPolicy` short-circuits repeated
  per-host transport/server failures. Open circuits surface through
  `NetworkError.underlying` wrapping `CircuitBreakerOpenError`. 4xx responses
  are state-aware: in `.closed` they are no-ops (the transport worked, the
  failure is semantic); in `.halfOpen` they release the probe slot and close
  the circuit; 5xx still counts as a transport failure.
- **Multipart response decoding** through `MultipartResponseDecoder` provides a
  buffered `multipart/*` parser for response bodies.
- **Optional macros** through the separate `Packages/InnoNetworkCodegen` package
  expose `@APIDefinition` and `#endpoint` while keeping `swift-syntax` out of
  the root runtime package graph.
- **Request construction contracts** now fix path joining, query handling,
  `Content-Type` attachment, and header precedence for signing-sensitive
  integrations.
- **Path encoding hardening** through the new public
  `EndpointPathEncoding.percentEncodedSegment(_:)` helper and the package
  `EndpointPathBuilder.makeURL` composer. Path literals containing raw
  whitespace or non-ASCII scalars are now percent-encoded automatically
  instead of failing `URLComponents.url` resolution and surfacing as
  `NetworkError.invalidRequestConfiguration`. Existing percent-escape
  sequences are preserved, and literals containing `?` or `#` still throw
  so query/fragment composition continues to flow through the dedicated
  encoders. Dynamic placeholder values supplied via `@APIDefinition` are
  routed through the segment helper so slashes inside an identifier no
  longer change the path hierarchy.
- **Retry idempotency** defaults to safe methods plus unsafe methods with an
  `Idempotency-Key`; method-agnostic retry is explicit opt-in. POST 503 +
  `Retry-After` + `Idempotency-Key` retries once with the same key; POST 503
  without `Idempotency-Key` does not retry; POST 503 without `Retry-After`
  but with `Idempotency-Key` falls through to exponential backoff.
- **Per-endpoint `acceptableStatusCodes`** lets endpoints treat codes such as
  `304 Not Modified` as success without changing the session-wide default.
- **`WebSocketCloseCode`** is part of the public WebSocket contract, including
  the full RFC 6455 range and application-defined `.custom(UInt16)` codes.
- **WebSocket heartbeat observability** is stable through `.ping`, `.pong`,
  and `.error(.pingTimeout)` events.
- **WebSocket reconnect counters** split into `attemptedReconnectCount` and
  `successfulReconnectCount`.
- **WebSocket bounded send queues** cap per-task in-flight send concurrency
  with `.fail` or `.dropNewest` overflow behavior.
- **Configurable WebSocket close handshake** via
  `WebSocketConfiguration.closeHandshakeTimeout: Duration` (default
  `.seconds(3)`, negative values clamp to `.zero`) replaces the previously
  hard-coded 3-second wait that follows `cancel(with:reason:)`.
- **Public-key pinning host matching** has an explicit strategy:
  `.unionAllMatches` covers exact and parent-domain pins as a union, while
  `.mostSpecificHost` isolates the exact host or longest matching parent.
- **`DownloadManager` is a public actor.** The 4.0.0 line removes the
  global `DownloadManager.shared` singleton; every feature constructs and
  owns a manager via `DownloadManager.make(configuration:)` (or the
  throwing initializer) with a unique session identifier. The throwing
  factory surfaces `DownloadManagerError.duplicateSessionIdentifier`
  directly, replacing the previous Optional/`fatalError` shapes.
- **Multipart and streaming readiness** includes `MultipartUploadStrategy`
  (defaults to `.streamingThreshold(bytes: 50 * 1024 * 1024)` for
  memory-bounded large uploads), `StreamingResumePolicy`,
  failure-payload redaction by default, and explicit persistence fsync
  policy.
- **Explicit streaming buffering policies**: every `AsyncStream` and
  `AsyncThrowingStream` factory owned by InnoNetwork declares a
  `bufferingPolicy`. Streaming responses, download delegate events, and
  event-hub consumer streams use `.unbounded` (no event loss); the
  `NetworkMonitor` path stream uses `.bufferingNewest(16)` so a slow
  observer only ever sees the most recent network state.
- **DecodingInterceptor** protocol exposes a `willDecode(data:response:)`
  hook that runs after response interceptors and before the configured
  decoder, plus a generic `didDecode(_:response:)` hook that observes the
  typed value before it returns to the caller. Both methods have default
  no-op implementations. `NetworkConfiguration.decodingInterceptors`
  registers them at the session level.
- **Response body limit** via `NetworkConfiguration.responseBodyLimit: Int64?`
  (default `nil`, opt-in) enforces a soft upper bound on buffered response
  bodies. Exceeding the limit short-circuits the decoder and throws
  `NetworkError/responseTooLarge(limit:observed:)`. Cache hits, conditional
  revalidation, and fresh transport responses are checked before cache
  writes; the final decoder input is rechecked after response and decoding
  interceptors. Endpoints that need genuine memory-bounded handling should
  use the streaming surface (`stream(_:)` / `bytes(for:)`).
- **Macro diagnostics with source locations**: `@APIDefinition` and
  `#endpoint` errors point at the offending argument, label, or placeholder
  rather than dropping every diagnostic on the macro attribute as a whole.
  IDE underlines now match the actual mistake.
- **DocC and operational docs** cover retry decisions, error classification,
  trust policies, background downloads, persistence, WebSocket close codes,
  reconnect, platform support, query encoding, production rollout, and
  cookbook articles for `WebSocketProtocolPolicy`,
  `WebSocketBackgroundTransition`, `ObservabilityExporters`,
  `OpenAPIGeneratorRecipe`, `DecodingInterceptorCookbook`,
  `Interceptors` (failure semantics), and `MigrationFromAlamofire`.
- **Release artifacts** are prepared with CycloneDX SBOM generation and
  sigstore cosign keyless signing.

## Stability Contract

The stable public API surface is defined in
[API_STABILITY.md](../../API_STABILITY.md). Lower-level request execution hooks
may exist in the source tree, but they are not part of the default `4.0.0`
stable contract.

No public generic execution-policy protocol is exposed in 4.0.0. The supported
public contract is the built-in policy surface on `NetworkConfiguration` and
`NetworkConfiguration.AdvancedBuilder`. Resilience policies default to
disabled/no-op behavior, and no new `NetworkError` enum case is added for open
circuit failures.

`InnoNetworkCodegen` is a separate compile-time package. Macro helpers require
`import InnoNetworkCodegen`; depending only on the root `InnoNetwork` package
does not resolve or build `swift-syntax`.

Swift OpenAPI Generator integration is supported as a recipe: wrap operations
as `APIDefinition` where possible, and treat SPI execution hooks as
revision-pinned generated-client support outside the stable import contract.
The `OpenAPIGeneratorRecipe` DocC article walks through the supported workflow.

## Migration

No migration is required for the additive changes. The one runtime behaviour
change worth flagging:

- **`NetworkError.responseTooLarge`**: `NetworkError` is a `public`
  non-`@frozen` enum, so the new `responseTooLarge(limit:observed:)` case
  requires consumers who write exhaustive `switch` statements over
  `NetworkError` to either handle the new case explicitly or add
  `@unknown default`. Code that catches `NetworkError` without exhaustive
  pattern matching is unaffected. The new case carries `errorCode == 4002`
  for `CustomNSError` bridging.
- **CircuitBreaker 4xx accounting**: a 4xx response no longer resets the
  consecutive-failure window. Consumers that were relying on the old 4xx
  reset behaviour to keep a host marked healthy were on an unstable
  contract — that path now matches the documentation.
- **MultipartAPIDefinition.uploadStrategy** now defaults to
  `MultipartUploadStrategy.platformDefault`, a memory-aware
  `streamingThreshold` (16 MiB on iOS/watchOS/tvOS, 50 MiB on
  macOS/visionOS). Endpoints that intentionally relied on the in-memory
  path should declare it explicitly:
  `var uploadStrategy: MultipartUploadStrategy { .inMemory }`.
- **`DownloadManager.shared`** is removed. Replace `DownloadManager.shared`
  call sites with an injected manager built via
  `DownloadManager.make(configuration:)` and a unique
  `sessionIdentifier`; route the failure (typically
  `DownloadManagerError.duplicateSessionIdentifier`) where it belongs in
  the owning feature.

## Deferred

- WebSocket `permessage-deflate` (RFC 7692) is deferred because
  `URLSessionWebSocketTask` does not expose deflate negotiation. The
  candidate is an optional `InnoNetworkWebSocketNIO` product so the existing
  URLSession-based WebSocket product stays stable.
- A persistent `ResponseCache` companion product is enumerated as an RFC
  (`docs/rfcs/persistent-response-cache.md`) — six policies (cache key
  composition, freshness precedence, eviction, privacy, data-protection
  class, on-disk format versioning) need to be decided before any
  implementation lands. The RFC positions persistent caching as a companion
  product rather than a core-module feature.
- A dedicated OpenAPI adapter package/product remains a roadmap candidate;
  the recipe-first path is documented in `OpenAPIGeneratorRecipe`.
- Pulse adapter example, streaming multipart response decoder, and
  Hummingbird in-process integration tests remain follow-up work.
- Future major-version candidates: `StateMachine` / `Reducer` protocol
  unification across Download, Refresh, and WebSocket lifecycles;
  lifecycle reducer generalization for Download and Refresh matching the
  WebSocket reducer model; persistent Response Cache companion product
  implementation; public Execution Policy protocol and an NIO transport
  adapter.
