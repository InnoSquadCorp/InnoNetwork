# InnoNetwork 4.0.0 Release Notes

InnoNetwork 4.0.0 is the first public release. The package targets Apple
platforms only and is built around Swift Concurrency, explicit transport
policies, and operational visibility from prototype to production.

## 한국어 요약

4.0.0은 최초 공개 릴리스이면서 49개 하드닝 항목을 같은 기준선에
흡수한 릴리스입니다. 보안/프라이버시, 캐시 정합성, 다운로드와
WebSocket 라이프사이클, HTTP 표준 준수, 성능 hot path, 테스트
인프라, CI supply chain을 한 번에 정리했습니다. 의도적인 breaking
change가 포함되어 있으므로 기존 사용자는
[`docs/Migration-4.0.0.md`](../Migration-4.0.0.md)의 즉시 마이그레이션
체크리스트를 먼저 확인해야 합니다.

주요 운영 기본값은 더 보수적으로 바뀌었습니다. 교차 origin redirect는

GitHub Actions는 full SHA로 pinning됐고, Dependabot과 dependency
review job이 PR supply-chain gate를 담당합니다.

## Highlights

- **Swift 6 language mode** is enabled for every package target.
- **`safeDefaults`** is the recommended configuration entry point for core,
  download, and WebSocket integrations.
- **Single transport entry point** on `APIDefinition`. Endpoints override
  `transport: TransportPolicy<APIResponse>` instead of six separate
  properties (`contentType`, `requestEncoder`, `queryEncoder`,
  `queryRootKey`, `decoder`, `responseDecoder`); the default is method-aware
  (`GET` → `.query()`, otherwise `.json()`).
- **Tag-based cancellation** through `CancellationTag`,
  `request(_:tag:)`, `upload(_:tag:)`, and `cancelAll(matching:)` so a
  screen, feature, or user session can drop just its own subset of
  in-flight requests.
- **Streaming pipeline extracted** into `StreamingExecutor`. Same observable
  behaviour as before but unit-testable on its own.
- **Stable request execution** is centered on `NetworkClient.request(_:)`
  and `NetworkClient.upload(_:)`.
- **Built-in resilience policies** add opt-in auth refresh, request
  coalescing, response caching, and circuit breaking. `RequestExecutionPolicy`
  is public for narrow custom transport-attempt policies.
- **Auth refresh** through `RefreshTokenPolicy` applies a current token,
  coordinates a single refresh across concurrent `401` responses, and replays
  each request at most once. The refresh-aware coalescer lane segregates
  callers entering during a refresh-in-flight window so a peer's pre-refresh
  transport result cannot leak into another caller's resumed continuation.
- **Request coalescing** through `RequestCoalescingPolicy` can fan out one raw
  transport result to identical in-flight requests. Authorization remains part
  of the deduplication key by default.
- **Response cache** through `ResponseCachePolicy`, `ResponseCache`,
  `CachedResponse`, `ResponseCacheKey`, and `InMemoryResponseCache` supports
  opt-in GET caching, ETag revalidation, `304` body substitution,
  stale-while-revalidate, locale separation, fragment stripping, and
  fingerprinted authorization keys. The pipeline also honours the response
  `Vary` header (RFC 9111 §4.1): `Vary: *` responses are not stored, and
  concrete `Vary` headers capture a request-header snapshot that future
  lookups must match. GET responses with cacheable whole-response status codes
  can be stored, `Cache-Control: no-store` and `Cache-Control: private`
  invalidate existing entries and skip writes, and `Cache-Control: no-cache`
  forces revalidation before reuse. Successful unsafe methods (`POST`, `PUT`,
  `PATCH`, `DELETE`, and unknown methods) invalidate all cached variants for
  the normalized target URI per RFC 9111 §4.4. A 304
  conditional revalidation that succeeds but carries a different `Vary`
  header preserves the stored entry's headers, body, and vary snapshot —
  the executor refreshes only `storedAt`. Responses to requests carrying
  `Authorization` are stored only when the origin explicitly permits it with
  `Cache-Control: public`, `must-revalidate`, or `s-maxage`.
- **Persistent response cache companion product** ships as
  `InnoNetworkPersistentCache`. `PersistentResponseCache` is an actor-backed
  `ResponseCache` with conservative defaults: 50 MB max size, 1000 entries,
  5 MB max entry, no authenticated response storage, no `Cache-Control:
  private` storage, no `Set-Cookie` response storage, and
  `.completeUnlessOpen` data protection. `dataProtectionClass: .none`
  requests `NSFileProtectionNone` for cache-owned paths, including existing
  index/body files on reopen. Sensitive disk keys are stored as managed
  HMAC-SHA256 values, App Group cache directories are supported, corrupt or
  unknown-version on-disk indexes self-heal by resetting the cache's own
  subtree, and statistics/telemetry snapshots expose scrub and eviction
  operations. Even when authenticated response storage is enabled, requests
  carrying `Authorization` require `Cache-Control: public`, `must-revalidate`,
  or `s-maxage`.
- **Circuit breaker** through `CircuitBreakerPolicy` short-circuits repeated
  per-host transport/server failures. Open circuits surface through
  `NetworkError.underlying` wrapping `CircuitBreakerOpenError`. 4xx responses
  are state-aware: in `.closed` they are no-ops (the transport worked, the
  failure is semantic); in `.halfOpen` they release the probe slot and close
  the circuit; 5xx still counts as a transport failure.
- **Multipart response decoding** through `MultipartResponseDecoder` provides a
  buffered `multipart/*` parser for response bodies. For large or long-lived
  responses, `MultipartStreamingResponseDecoder` emits ordered
  `partStarted`/`bodyChunk`/`partEnded` events from an async byte stream.
- **URL query array convention options** through
  `URLQueryArrayEncodingStrategy` support indexed brackets, empty brackets,
  and repeated keys while preserving indexed brackets as the default wire
  format.
- **Optional macros** through the separate `Packages/InnoNetworkCodegen` package
  expose `@APIDefinition` and `#endpoint` while keeping `swift-syntax` out of
  the root runtime package graph.
- **Request construction contracts** now fix path joining, query handling,
  `Content-Type` attachment, and header precedence for signing-sensitive
  integrations.
- **Redirect and insecure-HTTP guards** are secure by default:
  `DefaultRedirectPolicy` strips credential-bearing headers across origins,
  plain HTTP base URLs require `allowsInsecureHTTP = true`, and embedded URL
  credentials or fragments are rejected before dispatch.
- **Cache-Control parsing** is quoted-string aware, so privacy directives
  such as `private="Set-Cookie, Authorization"` still invalidate matching
  cache entries and skip storage.
- **Path encoding hardening** through the new public
  `EndpointPathEncoding.percentEncodedSegment(_:)` helper and the package
  `EndpointPathBuilder.makeURL` composer. Path literals containing raw
  whitespace or non-ASCII scalars are now percent-encoded automatically
  instead of failing `URLComponents.url` resolution and surfacing as
  `NetworkError.configuration(reason: .invalidRequest(...))`. Existing
  percent-escape sequences are preserved, and literals containing `?` or
  `#` still throw so query/fragment composition continues to flow through
  the dedicated encoders. Dynamic placeholder values supplied via
  `@APIDefinition` are routed through the segment helper so slashes inside
  an identifier no longer change the path hierarchy.
- **Retry idempotency** defaults to safe methods (`GET`, `HEAD`, `OPTIONS`,
  `TRACE`) plus unsafe methods with an `Idempotency-Key`; method-agnostic retry
  is explicit opt-in. POST 503 +
  `Retry-After` + `Idempotency-Key` retries once with the same key; POST 503
  without `Idempotency-Key` does not retry; POST 503 without `Retry-After`
  but with `Idempotency-Key` falls through to exponential backoff.
- **Automatic idempotency keys** through `IdempotencyKeyPolicy` attach one
  stable key per logical request so retries reuse the same value instead of
  re-running a plain interceptor.
- **Production preset** through `NetworkConfiguration.recommendedForProduction`
  enables conservative retry, circuit breaker, and idempotency-key defaults
  while keeping auth, cache storage, and custom execution policies opt-in.
- **W3C Trace Context** through `TraceContextInterceptor` propagates existing
  `traceparent` values or creates a standards-compliant header without adding
  an OpenTelemetry dependency to the runtime package.
- **Curl export** through `URLRequest.curlCommand(...)` renders redacted,
  shell-escaped reproduction commands for debugging.
- **Localized error descriptions** for `NetworkError` ship with English and
  Korean catalogues while preserving the existing English fallback wording.
- **DocC tutorial** "Build a GitHub Client" shows the minimal path from a
  typed `APIDefinition` to `DefaultNetworkClient.request(_:)`.
- **Benchmark memory snapshots** report process resident high-water marks and
  current resident deltas as observational signals; throughput remains the
  4.0.0 regression guard.
- **Streaming output buffering** can now be bounded with
  `stream(_:bufferingPolicy:)` for high-volume streams that prefer capped
  memory over lossless decoded output. Keep the default lossless `stream(_:)`
  for SSE feeds that rely on `StreamingResumePolicy.lastEventID`; the client
  rejects that resume policy with bounded buffers because they can intentionally
  drop decoded values before the consumer processes them.
- **Per-endpoint `acceptableStatusCodes`** lets endpoints treat codes such as
  `304 Not Modified` as success without changing the session-wide default.
- **`WebSocketCloseCode`** is part of the public WebSocket contract, including
  the full RFC 6455 range and application-defined `.custom(UInt16)` codes.
- **WebSocket heartbeat observability** is stable through `.ping`, `.pong`,
  and `.error(.pingTimeout)` events.
- **WebSocket reconnect counters** split into `attemptedReconnectCount` and
  `successfulReconnectCount`.
- **WebSocket bounded send queues** cap per-task in-flight send concurrency
  with `.fail` or `.dropNewest` overflow behavior.
- **Configurable WebSocket close handshake** via
  `WebSocketConfiguration.closeHandshakeTimeout: Duration` (default
  `.seconds(3)`, negative values clamp to `.zero`) replaces the previously
  hard-coded 3-second wait that follows `cancel(with:reason:)`.
- **Public-key pinning host matching** has an explicit strategy:
  `.unionAllMatches` covers exact and parent-domain pins as a union, while
  `.mostSpecificHost` isolates the exact host or longest matching parent.
- **`DownloadManager` is a public actor.** The 4.0.0 line removes the
  global `DownloadManager.shared` singleton; every feature constructs and
  owns a manager via `DownloadManager.make(configuration:)` (or the
  throwing initializer) with a unique session identifier. The throwing
  factory surfaces `DownloadManagerError.duplicateSessionIdentifier`
  directly, replacing the previous Optional/`fatalError` shapes.
- **Download restoration contract is public and observable**:
  `waitForRestoration()` exposes the restore gate, missing system tasks emit
  `.failed(.restorationMissingSystemTask)`, foreign system tasks are cancelled,
  and paused records with `resumeData` can be restored after process restart.
- **Download append-log compaction** is configurable through
  `DownloadConfiguration.PersistenceCompactionPolicy`.
  Checkpoints written before the optional `orderedRecordIDs` snapshot field
  cannot perfectly reconstruct the latest task id for repeated same-URL
  records; restoration uses `orderedRecordIDs` first when present, then appends
  unseen records in sorted-ID order as a deterministic best-effort fallback
  until a fresh checkpoint is written.
- **Multipart and streaming readiness** includes `MultipartUploadStrategy`
  (defaults to `.platformDefault`: 16 MiB on iOS/watchOS/tvOS,
  50 MiB on macOS/visionOS), `StreamingResumePolicy`,
  failure-payload redaction by default, and explicit persistence fsync
  policy.
- **Explicit streaming buffering policies**: every `AsyncStream` and
  `AsyncThrowingStream` factory owned by InnoNetwork declares a
  `bufferingPolicy`. Streaming responses, download delegate events, and
  event-hub consumer streams use `.unbounded` (no event loss); the
  `NetworkMonitor` path stream uses `.bufferingNewest(16)` so a slow
  observer only ever sees the most recent network state.
- **DecodingInterceptor** protocol exposes a `willDecode(data:response:)`
  hook that runs after response interceptors and before the configured
  decoder, plus a generic `didDecode(_:response:)` hook that observes the
  typed value before it returns to the caller. Both methods have default
  no-op implementations. `NetworkConfiguration.decodingInterceptors`
  registers them at the session level.
- **Streaming-by-default inline responses** use `URLSession.bytes(for:)` plus
  bounded collection before decode. `ResponseBodyBufferingPolicy` owns the
  buffering mode and max-byte guard; `responseBodyLimit` remains a
  compatibility alias.
- **Public reducer vocabulary** through `StateReducer` and `StateReduction`
  standardizes lifecycle decision functions. WebSocket and Download lifecycle
  decisions now share the same reducer shape internally.
- **Phantom auth scopes** let endpoints declare `AuthRequiredScope`.
  `EndpointBuilder<Response, AuthRequiredScope>` fails before transport unless
  `NetworkConfiguration.refreshTokenPolicy` is configured.
- **Macro diagnostics with source locations**: `@APIDefinition` and
  `#endpoint` errors point at the offending argument, label, or placeholder
  rather than dropping every diagnostic on the macro attribute as a whole.
  IDE underlines now match the actual mistake.
- **Macro optional placeholder rejection**: `@APIDefinition` fails at compile
  time when a path placeholder maps to an optional stored property.
- **DocC and operational docs** cover retry decisions, error classification,
  trust policies, background downloads, persistence, WebSocket close codes,
  reconnect, platform support, query encoding, production rollout, and
  cookbook articles for `WebSocketProtocolPolicy`,
  `WebSocketBackgroundTransition`, `ObservabilityExporters`,
  `OpenAPIGeneratorRecipe`, `DecodingInterceptorCookbook`,
  `Interceptors` (failure semantics), and `MigrationFromAlamofire`.
- **Benchmark trend automation** adds JSON baseline deltas, PR-comment
  markdown rendering, scheduled JSONL history on the `benchmark-trends`
  branch, and baseline rationale tracking.
- **Release artifacts** are prepared with CycloneDX SBOM generation and
  sigstore cosign keyless signing.
- **Supply-chain CI** includes SwiftPM/GitHub Actions Dependabot updates,
  full-SHA-pinned workflow actions, and pull-request dependency review.

## release/4.0.0-batch additions

The final pre-cut batch landed on top of the original 49-item hardening
pass and tightened several long-standing design seams. See `CHANGELOG.md`
for the per-commit list and `docs/Migration-4.0.0.md` for migration
notes; the highlights are:

- **`WebSocketManager` is now an `actor`.** `await` is required at every
  public call site. The four URLSession delegate-bridge entry points and
  `handleBackgroundSessionCompletion` stay `nonisolated` so synchronous
  URLSession callbacks keep working.
- **`NetworkError` adds `.transportSuspended` and
  `.cacheRevalidationFailed(underlying:cached:)`.**
  `.transportSuspended` is emitted when `.requiresConnection` persists
  through the reachability policy wait, distinct from
  `.configuration(reason: .offline(...))`; cache revalidation failure is
  distinct from `.underlying`/`.decoding`. Localized strings ship in
  `en` and `ko`.
- **`StreamingResumeStrategy` protocol.** `StreamingResumePolicy`
  retroactively conforms; the streaming bounded-buffer guard now flows
  through `isCompatible(with:)` rather than pattern-matching the enum
  directly. Future strategies plug in without inline executor changes.
- **NetworkConfiguration fluent modifiers.** Seven `.with(...)`
  modifiers remain as deprecated migration helpers. New code should use
  the pack-based `advanced(baseURL:resilience:auth:observability:cache:transport:)`
  factory so policy precedence stays visible.
- **Persistent cache hit/miss/eviction counters** exposed on
  `PersistentResponseCacheStatistics`. The eviction counter seeds from
  the open-time scrub pipeline so it reflects the entire actor lifetime.
- **`HTTPHeaderName<Variant>` phantom-typed keys** for single-value
  versus repeatable headers, alongside the existing string-keyed
  surface.
- **DownloadTask gains `generation` / `attempt` accessors** that mirror
  the WebSocket lifecycle's epoch tracking so retry-cycle callbacks can
  be disambiguated without exposing lifecycle mutation as public API.
- **Documentation:** five-policy compatibility matrix in
  `docs/PolicyInteractions.md` and an RFC 9111 compliance matrix at
  `docs/rfcs/RFC9111-Compliance.md`. The `Tools/openapi-to-innonetwork`
  package gains a 5.0 design note for the planned
  `apple/swift-openapi-generator` transport adapter.

## Stability Contract

The stable public API surface is defined in
[API_STABILITY.md](../../API_STABILITY.md). Lower-level request execution hooks
may exist in the source tree, but they are not part of the default `4.0.0`
stable contract.

`RequestExecutionPolicy` is part of the public 4.0.0 contract for custom
transport-attempt behavior. Built-in retry, auth refresh, coalescing, cache,
and circuit-breaker semantics remain first-class configuration surfaces on
`NetworkConfiguration` and `NetworkConfiguration.AdvancedBuilder`. Resilience
policies default to disabled/no-op behavior, and no new `NetworkError` enum
case is added for open circuit failures.

`InnoNetworkCodegen` is a separate compile-time package. Macro helpers require
`import InnoNetworkCodegen`; depending only on the root `InnoNetwork` package
does not resolve or build `swift-syntax`.

Swift OpenAPI Generator integration is provided through the
`InnoNetworkOpenAPI` companion product. Wrap each generated operation as an
`OpenAPIRestOperation` and dispatch it as an `OpenAPIRequest` through any
`NetworkClient`. The SPI execution hooks remain supported, but they are
revision-pinned to the generated client they back and sit outside the stable
import contract that governs `OpenAPIRestOperation` and `OpenAPIRequest`. The
`OpenAPIGeneratorRecipe` and `OpenAPIGeneratorAdapter` DocC articles walk
through the workflow end to end.

## Migration

4.0.0 includes breaking API and behavior changes by design. Migration items:

- **`NetworkError.responseTooLarge`**: `NetworkError` is a `public`
  non-`@frozen` enum, so the new `responseTooLarge(limit:observed:)` case
  requires consumers who write exhaustive `switch` statements over
  `NetworkError` to either handle the new case explicitly or add
  `@unknown default`. Code that catches `NetworkError` without exhaustive
  pattern matching is unaffected. The new case carries `errorCode == 4002`
  for `CustomNSError` bridging.
- **CircuitBreaker 4xx accounting**: a 4xx response no longer resets the
  consecutive-failure window. Consumers that were relying on the old 4xx
  reset behaviour to keep a host marked healthy were on an unstable
  contract — that path now matches the documentation.
- **MultipartAPIDefinition.uploadStrategy** now defaults to
  `MultipartUploadStrategy.platformDefault`, a memory-aware
  `streamingThreshold` (16 MiB on iOS/watchOS/tvOS, 50 MiB on
  macOS/visionOS). Endpoints that intentionally relied on the in-memory
  path should declare it explicitly:
  `var uploadStrategy: MultipartUploadStrategy { .inMemory }`.
- **`DownloadManager.shared`** is removed. Replace `DownloadManager.shared`
  call sites with an injected manager built via
  `DownloadManager.make(configuration:)` and a unique
  `sessionIdentifier`; route the failure (typically
  `DownloadManagerError.duplicateSessionIdentifier`) where it belongs in
  the owning feature.
- **Inline request responses stream by default**. Code that depended on
  `URLSession.data(for:)`-style eager buffering can set
  `responseBodyBufferingPolicy = .buffered(maxBytes:)`; code that only used
  `responseBodyLimit` can keep doing so through the compatibility alias.
- **Auth-required endpoints need a refresh policy**.
  `EndpointBuilder<Response, AuthRequiredScope>` and custom `APIDefinition`
  types with `typealias Auth = AuthRequiredScope` fail before transport when
  `refreshTokenPolicy` is absent.
- **Fluent endpoint names are finalized**. Replace `Endpoint<Response>` with
  `EndpointBuilder<Response, PublicAuthScope>`, replace
  `AuthenticatedEndpoint<Response>` with
  `EndpointBuilder<Response, AuthRequiredScope>`, and replace
  `ScopedEndpoint<Response, Scope>` with `EndpointBuilder<Response, Scope>`.
  Builder entry points use
  `EndpointBuilder<EmptyResponse, PublicAuthScope>.get(...)` or
  `EndpointBuilder<EmptyResponse, AuthRequiredScope>.get(...)` explicitly.

## Deferred

- Full WebSocket `permessage-deflate` (RFC 7692) negotiation is still outside
  the URLSession transport because `URLSessionWebSocketTask` does not expose
  deflate negotiation. 4.0.0 now fails `permessageDeflateEnabled` on
  URLSession with `WebSocketError.unsupportedProtocolFeature(.permessageDeflate)`
  instead of silently ignoring it. A real NIO-backed transport remains a
  follow-up optional package so the core request product stays URLSession-first.
- Pulse adapter example and Hummingbird in-process integration tests remain
  follow-up work.
- Future minor/major candidates: richer OpenAPI/HTTPTypes adapters that pin
  external generator versions, public Execution Policy protocol expansion, and
  an NIO transport adapter. Refresh, Download, and WebSocket now share reducer
  ownership patterns where it materially reduces lifecycle regression risk.

### Hardening Pass — Breaking

- `URLQueryEncoder`: `nonConformingFloatEncodingStrategy` defaults to
  `.throw`. NaN/Infinity now raise `EncodingError.unsupportedValue(reason:)`
  instead of serializing to provider-dependent strings. `Decimal` values use
  `Decimal.description` so non-en_US locales no longer emit `,` decimal
  separators. `Data` remains standard Base64 via `Data.base64EncodedString()`,
  and `encodeForm` produces RFC 1866 `application/x-www-form-urlencoded`
  output (space → `+`, `+` percent-encoded).
- `HTTPHeader`: storage is now an ordered list. `Set-Cookie` and
  `WWW-Authenticate` retain duplicate values; the dictionary projection
  comma-joins repeated case-insensitive names while preserving the first
  spelling as the canonical key.
- `MultipartFormData.appendFile(at:)` is now `throws` (was `async throws`)
  and validates file existence at append time. `encode()` surfaces file-read
  failures rather than silently dropping parts. RFC 5987 `filename*=UTF-8''…`
  is emitted for non-ASCII filenames; ASCII fallback is preserved.
- `MultipartResponseDecoder`: missing/invalid boundary raises
  `NetworkError.configuration(reason: .invalidRequest(...))` instead of
  returning an empty array.
- `RetryPolicy.init`: gains `jitterFactor` and `maxTotalRetryDuration`
  parameters with safe defaults. The `cancelled` event now fires even when
  the surrounding task is cancelled.
- `RefreshTokenPolicy`: refresh completion now drives the
  idle/in-flight/cooldown state from the detached refresh task itself, so
  caller cancellation while awaiting a refresh no longer clears single-flight
  state or blocks the cancelled awaiter until the shared refresh finishes.
  Releasing the coordinator while a refresh is still in flight cancels the
  orphan task. Consecutive failures enter
  `RefreshFailureCooldown.exponentialBackoff(base: 1.0, max: 30.0)`; callers
  during cooldown receive the cached error rather than triggering a hot-loop
  refresh. `Authorization` strip is case-insensitive.
- `CircuitBreakerPolicy.init(validatedFailureThreshold:windowSize:resetAfter:maxResetAfter:numberOfProbesRequiredToClose:countsTransportSecurityFailures:)`
  adds explicit throwing validation while the existing
  `init(failureThreshold:windowSize:...)` remains source-compatible and
  silently clamps. Keys are derived from `scheme://host:port` so different
  ports are isolated. The state machine uses a true rolling window and
  supports configurable hysteresis via `numberOfProbesRequiredToClose`.
  TLS pinning and certificate trust failures are excluded from the failure
  count by default; DNS/name-resolution failures remain regular underlying
  transport failures.
- `DownloadConfiguration.safeDefaults` and `advanced` set

### Hardening Pass — Fixed

- Redirect handling no longer leaks credential-bearing headers across
  origins, and base URLs containing embedded `user:password@` credentials
  or fragments fail before transport dispatch.
- `Cache-Control` parsing is quoted-string aware. Directives such as
  `private="Set-Cookie, Authorization"` now still count as `private`,
  invalidating the current cache key and skipping storage.
- `URLQueryEncoder`: `SnakeCaseKeyTransformCache` is bounded to 4096
  entries to prevent unbounded growth on dynamic key sets.
- `RetryCoordinator`: catch branches are deduplicated, finish ordering is
  awaited (no detached `Task` for the terminal event), and the `unknown`
  error path retains request context. Retry-After is documented as a floor.
- `ResponseCachePolicy`: query items are sorted before fingerprinting so
  semantically identical URLs hit the same cache entry. `Cookie`,
  `Proxy-Authorization`, `X-Api-Key`, and `X-Auth-Token` are sensitive by
  default. The in-memory LRU is now O(1) (doubly-linked list + dict);
  `byteCost` includes URL/method/varyHeaders/storedAt; `cachedResponseMatchesVary`
  trims OWS and treats `Accept-Encoding` as a token set. The
  `rfc9111Compliant(wrapping:)` adapter now falls back to `Expires` freshness
  when no valid `max-age` exists, then to RFC 9111 §4.2.2 `Last-Modified`
  heuristic freshness when neither exists, and treats invalid freshness
  metadata as stale. Persistent cache disk keys now include the `Vary`
  snapshot so variants no longer overwrite each other.
- `WebSocketReconnectCoordinator`: any prior reconnect task is cancelled
  before a new one is registered. `URLError.cannotConnectToHost`,
  `.networkConnectionLost`, `.notConnectedToInternet`, and `.cancelled`
  are classified for ping-timeout handling. Backoff guards against
  `pow(2, -1)` and inverted random ranges. `WebSocketConfiguration`
  exposes `maximumMessageSize`, `permessageDeflateEnabled`, and
  `reconnectMaxTotalDuration`.
- `DownloadTaskPersistence`: `id(forURL:)` is O(1) via a maintained reverse
  index. The append log is replayed via `FileHandle` chunk-streaming so
  memory stays bounded on multi-MB logs. `withDirectoryLock` polls
  `flock(LOCK_EX | LOCK_NB)` with a 10s deadline and 50ms backoff instead
  of blocking indefinitely. The `fileManager` parameter is now actually
  honored. The persisted `Record` schema remains `id`/`url`/`destinationURL`/
  `resumeData`.
- `TrustPolicy`: `SecTrustEvaluateWithError` captures the underlying error
  and surfaces it via `NetworkError.trustEvaluationFailed(...)`.
- `NetworkLogger`: JWT-shaped tokens are auto-masked. CLI environments can
  inspect the same redacted payload through `os_log`.
- `EventDeliveryPolicy.default` is `.dropOldest(buffer: 256)`; unbounded
  buffering is an explicit opt-in.
- `NetworkMonitor` exposes explicit `start()`/`stop()` and cancels its
  `pathUpdateHandler` on `deinit`.
- `InFlightRegistry` cancels the underlying `URLSessionTask` when
  `cancelAll(matching:)` fires, so tag-based cancellation drops the wire
  in milliseconds.
- `URLRequest.headers` setter routes per-header through `setValue`/`addValue`
  instead of the dictionary projection so multi-value entries
  (`WWW-Authenticate`, etc.) survive round-tripping into a request.
- `PersistentResponseCache`: `lastAccessedAt` updates on the read path skip
  the durability `fsync` even under `.always` so cache-hit latency does not
  amplify into per-read disk barriers. LRU eviction is now a single sort
  + drain (was O(N²) on bulk overflow).
- `RefreshTokenCoordinator`: state transitions (idle/inFlight/cooldown) are
  driven by the detached refresh task itself rather than by the awaiter's
  catch arms, preserving single-flight even under aggressive caller
  cancellation. Awaiting callers now observe their own cancellation promptly
  without cancelling a refresh shared with other callers.
- `RetryCoordinator`: cancellation event publishing is unified at a single
  chokepoint in `execute(...)` so all three catch arms produce exactly one
  `.requestFailed` event for cancelled requests.
- `MultipartFormData`: non-ASCII `name=` parts emit a paired `name*=UTF-8''…`
  RFC 5987 companion alongside the ASCII fallback (matching the existing
  `filename*` behaviour) so receivers that understand the extended syntax
  recover the original UTF-8 bytes.
- `WebSocketHeartbeatCoordinator`: every failed ping publishes
  `.error(.pingTimeout)` regardless of whether the underlying error
  matches the heartbeat classifier — silent unclassified errors no longer
  hide mid-link failures until the missed-pong threshold trips.
- `DefaultNetworkClient`: `DefaultNetworkClient(configuration:)` constructs a
  fresh session from `makeURLSessionConfiguration()` with per-client cookie
  storage and an in-memory URL cache. Process-wide `URLSession.shared` state is
  still available by passing it explicitly.
- `DownloadTaskPersistence`: `mutate(...)` acquires the directory lock via
  `Task.sleep`-based polling so a contended lock no longer pins a
  cooperative-executor thread under `usleep`.

### Removed

- `DownloadManager.shared` is removed in 4.0.0. The previous accessor
  trapped on first access in failure modes (duplicate session identifier,
  unavailable persistence) and forced every feature onto a single
  `DownloadConfiguration`. Construct managers via
  `DownloadManager.make(configuration:)` with a unique session identifier
  per feature; the throwing factory surfaces `DownloadManagerError`
  directly so callers can react to `duplicateSessionIdentifier` instead
  of receiving an Optional or trapping.
- `NetworkError.objectMapping(_:_:)` static factory is removed. Decode
  failures now surface exclusively as
  `NetworkError.decoding(stage:underlying:response:)` with a
  `DecodingStage` (`.responseBody`, `.streamFrame`) so retry policies
  and observability layers can distinguish where in the pipeline the
  failure happened. Migrate
  pattern matching from `.objectMapping(let underlying, let response)`
  to `.decoding(let stage, let underlying, let response)`; new
  `NetworkError.isDecodingFailure` is the canonical helper for "decode
  failures are not retried".
- `MultipartFormData.appendFile(at:name:mimeType:) throws` (the
  synchronous in-memory overload) is removed. Use the async overload
  combined with `writeEncodedData(to:)` so file bytes stream from disk
  without loading into memory at append time.
- `NetworkError.undefined` and `NetworkError.jsonMapping` — both cases were
  unreachable in production code and only existed as test fixtures.
  Decode failures now surface as `.decoding(stage:underlying:response:)`.

### Fixed

- **PersistentResponseCache overwrite (PR #39)**: `set()` on an existing key
  no longer deletes the freshly-written body file. The bodyfile cleanup runs
  only when the new entry resolves to a different filename than the existing
  one, so subsequent `get()` calls hit the new payload instead of seeing a
  cold cache. `get()`'s body-read failure handling is now separated from the
  best-effort `lastAccessedAt` index write, so a transient index write
  failure (e.g. read-only mount) no longer demotes a successful read to a
  miss.
- **PersistentResponseCache recovery scope (PR #39)**: index recovery on
  unknown version or decode failure now deletes only the cache's own
  `index.json` and `bodies/` subtree. The user-supplied configuration
  directory itself is preserved, so adjacent files in a shared parent
  directory (`sentinel.txt`, sibling caches) survive recovery.
- **RequestExecutor `responseReceived` placement (PR #39)**: the
  `NetworkEvent.responseReceived` event now fires inside the transport
  boundary used by custom `RequestExecutionPolicy` chains. A policy that
  calls `next.execute(_:)` more than once now produces one event per
  transport attempt; a policy that returns a synthetic response without
  calling `next` no longer emits a synthetic transport event.
- **Streaming `maxBytes` fallback (PR #39)**: when
  `responseBodyBufferingPolicy` is `.streaming(maxBytes:)` and the transport
  reports `invalidRequestConfiguration`, the executor no longer falls back
  to the buffered `data(for:)` path. The configured byte cap is honoured —
  the request fails fast instead of silently buffering an unbounded body.
- **Multipart auth scope (PR #39)**: `MultipartAPIDefinition` now declares
  `associatedtype Auth: AuthScope = PublicAuthScope`, so a multipart
  endpoint conforming to `AuthRequiredScope` participates in the configured
  `RefreshTokenPolicy` exactly like a non-multipart authenticated endpoint.
  Default behaviour is unchanged (`PublicAuthScope`).
- **Download resume-data clear (PR #39)**: `DownloadManager.resume(_:)` now
  surfaces persistence failures when clearing the stored `resumeData`
  before starting the new system task. On failure the in-flight task is
  cancelled and the task is surfaced as `.failed(.persistenceFailure)`,
  preventing a resumed transfer from running with stale resume bytes still
  on disk.
- **Restore-missing event delivery (PR #39)**: the
  `.failed(.restorationMissingSystemTask)` failure for orphaned persisted
  records is no longer published from `DownloadManager.init`. The event is
  flushed when the caller subscribes via `events(for:)`, so the failure is
  observable instead of being dropped into a partition with no consumer.
- **CircuitBreaker (P1.1)**: `CircuitBreakerRegistry.recordStatus(...)` no
  longer resets the rolling failure window on 4xx responses. 4xx is now a
  no-op (the transport worked, the failure is semantic), 5xx still counts
  as a transport failure, and 2xx/3xx clear the accumulated failures and
  release any half-open probe slot. Previously a single 4xx interleaved
  between 5xx/timeout failures could mask a host that was teetering on
  the failure threshold. `CircuitBreakerRegistry.recordStatus` now also
  releases the half-open probe slot when the probe response is a 4xx —
  in `.halfOpen` a 4xx closes the circuit because the probe purpose is
  to confirm the transport works, and a 4xx response satisfies that.
- **Streaming back-pressure (P1.7)**: every async stream factory now
  declares a `bufferingPolicy` explicitly. Streaming response bytes,
  download delegate events, and event-hub consumer streams use
  `.unbounded` (event loss would corrupt task lifecycles or drop
  server-emitted records); `NetworkMonitor` snapshots use
  `.bufferingNewest(16)` because only the latest path state is meaningful.
- **WebSocket close handshake timeout (P1.12)**:
  `WebSocketManager.disconnect(_:closeCode:)` now snapshots
  `configuration.closeHandshakeTimeout` into a local before spawning the
  close-handshake timer. The previous code read the timeout via
  `self?.configuration.closeHandshakeTimeout ?? .seconds(3)` *inside* the
  spawned task, so a deallocated manager would still sleep the default
  3 seconds before the no-op tail and would silently fall back to the
  default even when callers had configured a non-default value.
- **WebSocket lifecycle table (P2.8)**:
  `WebSocketState.connecting.nextStates` now lists `.reconnecting`. The
  manager already drives a connecting → reconnecting transition when a
  handshake fails and the close disposition allows reconnect, so the
  documented transition table now matches the runtime. No behavioural
  change.
- **Macro diagnostics (P1.14)**: `InnoNetworkMacros` diagnostics emitted by
  `@APIDefinition` and `#endpoint` now attach their `SourceLocation` to
  the offending syntax node (the missing argument list, the non-literal
  `path:` expression, the unlabeled `as:` argument, etc.) instead of
  falling back to a generic location on the macro attribute. IDE squiggles
  and `swift build` diagnostics now point at the exact argument the user
  has to fix. `MacroDiagnostic` exposes a new `error(at: some SyntaxProtocol)`
  helper that wraps the diagnostic in a `DiagnosticsError` anchored at the
  supplied node; `APIDefinitionMacro` and `EndpointMacro` thread the most
  precise syntax node through every throw site.
- **TimeoutReason mapping (doc + tests)**: `TimeoutReason` documentation now
  enumerates the exact `URLError` → `NetworkError.timeout(reason:)` mapping
  (`.timedOut` → `.requestTimeout`, `.cannotConnectToHost` →
  `.connectionTimeout`) and records why other transport codes (DNS failures,
  reachability errors, mid-flight drops) intentionally surface as
  `NetworkError.underlying` instead of being reclassified as timeouts.
  Lock-down tests cover `URLError.cancelled`, `.networkConnectionLost`,
  and `.notConnectedToInternet`.
- **DownloadTask state guard**: `DownloadTask.updateState(_:)` now validates
  transitions against the documented `DownloadState.canTransition(to:)` table.
  Illegal transitions trigger `assertionFailure` in DEBUG and an OSLog `.fault`
  in release, then reject the assignment so the existing state is preserved.
  State restoration (rebuilding actor state from persisted records or live
  `URLSession` task state on relaunch) and test-only state injection now
  use the new `restoreState(_:)` helper, which bypasses validation.
- **Refresh-aware coalescer lanes (P2.1)**: when a refresh is in flight,
  callers entering ``RequestExecutor`` now receive a unique coalescer
  lane suffix so a peer's pre-refresh transport result cannot leak into
  another caller's resumed continuation. Under the default coalescing
  policy (``Authorization`` participates in the dedup key) this is a
  defence-in-depth pin; under a policy that excludes ``Authorization``
  from the key it is the actual safeguard. ``RefreshTokenCoordinator``
  now exposes ``isRefreshInProgress`` (package-visible, actor-isolated)
  so the executor can read the lane state without breaking encapsulation.
- **Response cache 304-with-new-Vary preservation (P1.2)**: a 304 Not
  Modified response that carries a `Vary` header different from the one
  that was active when the cached entry was stored no longer rewrites
  the entry under the new vary dimension. The executor now refreshes the
  existing entry's `storedAt` while preserving its headers, body, and
  vary snapshot — a 304 confirms freshness of the stored representation,
  but the 304's own `Vary` describes the variant the origin would have
  served on a full 200 and must not silently rekey the stored entry.
  Behaviour is unchanged when the 304 carries the same `Vary` header
  (or no `Vary` header), so existing call sites are unaffected.

### Concurrency

- `DownloadManager` is a `public actor`. URL-session delegate callbacks
  enqueue into one delegate-event stream that a single consumer drains into
  the actor. `handleBackgroundSessionCompletion(_:completion:)` is
  `nonisolated` so the synchronous Foundation entry point stays usable.
- `DownloadConfiguration.persistenceFsyncPolicy: PersistenceFsyncPolicy`
  picks one of `.always`, `.onCheckpoint` (default), `.never` for the
  append-log durability barrier. The store calls `Darwin.fsync(_:)` after
  append-log mutation batches (only `.always`) and checkpoint writes (every
  policy except `.never`). See
  [`Sources/InnoNetworkDownload/InnoNetworkDownload.docc/Articles/Persistence.md`](../../Sources/InnoNetworkDownload/InnoNetworkDownload.docc/Articles/Persistence.md).
- No `@unchecked Sendable` usages in production sources. CI rejects any new
  `@unchecked Sendable` in shipping library targets.

### Migration

`NetworkError` is a `public` non-`@frozen` enum, so the new
`responseTooLarge(limit:observed:)` case requires consumers who write
exhaustive `switch` statements over `NetworkError` to either handle the
new case explicitly or add `@unknown default`. Code that catches
`NetworkError` without exhaustive pattern matching is unaffected. The
new case carries `errorCode == 4002` for `CustomNSError` bridging.

### Deferred

- WebSocket `permessage-deflate` extension (RFC 7692) requires a transport
  substitution because `URLSessionWebSocketTask` does not expose deflate
  negotiation. The candidate is an optional `InnoNetworkWebSocketNIO`
  product on swift-nio so the URLSession-based product stays stable.
- OpenAPI Generator integration is recipe-first. The supported path is the
  `APIDefinition` wrapper documented in DocC; generated-client SPI hooks
  remain outside the default stable contract. A separate adapter
  package/product is tracked as a roadmap candidate.
- Pulse adapter example, streaming multipart decoder, and Hummingbird
  in-process integration tests remain follow-up work.
