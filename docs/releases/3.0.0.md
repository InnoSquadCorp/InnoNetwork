# InnoNetwork 3.0.0 Release Notes

## Summary

InnoNetwork 3.0.0 was an earlier public OSS release for the framework family:

- `InnoNetwork`
- `InnoNetworkDownload`
- `InnoNetworkWebSocket`

This release turns the package from an internal framework into a documented public SDK with an explicit stability contract, release policy, support posture, benchmark governance, and consumer smoke validation.

## Highlights

### Public SDK contract

- Introduced a formal public API compatibility document in [API_STABILITY.md](../../API_STABILITY.md)
- Standardized semver and release expectations for the earlier release line
- Added OSS governance files for contribution, support, security, and changelog management

### Safer configuration surface

- Promoted `safeDefaults` as the recommended entry point for public consumers
- Kept `advanced` builders for explicit tuning without making internal policy types part of the public contract

### Runtime and operational improvements

- Added deterministic request encoding and explicit decoding strategies
- Improved event delivery with bounded buffering, consumer isolation, and telemetry
- Replaced lightweight download persistence with append-log durability
- Refined websocket reconnect behavior with handshake-aware classification and reconnect suppression

### Tooling and release quality

- Added docs / contract sync automation
- Added documentation smoke and external consumer smoke validation
- Added benchmark governance with machine-readable output and baseline diff reporting

## Upgrade Notes

- Public examples and package installation instructions targeted `3.0.0` at
  the time of this release
- Current repository documentation targets the latest `4.0.0` public release
  line
- Internal policy layers such as `RequestEncodingPolicy`, `ResponseDecodingStrategy`, reconnect taxonomy internals, and persistence file formats remain non-contract implementation details

## Validation

Release validation for 3.0.0 includes:

- `swift test`
- docs / contract sync
- documentation smoke build and run
- benchmark target smoke build and quick benchmark run
- external consumer smoke build
