# InnoNetwork 3.0.1 Release Notes

## Summary

InnoNetwork 3.0.1 is the current public OSS release line for the framework family:

- `InnoNetwork`
- `InnoNetworkDownload`
- `InnoNetworkWebSocket`

This patch release keeps the public `3.x` contract stable while tightening release guidance, consumer-facing examples, and OSS delivery policy around the current version line.

## Highlights

### Public SDK contract

- Keeps the `3.x` semver line as the stable public contract
- Continues to treat `safeDefaults` as the recommended public entry point
- Preserves the documented stability boundary in [API_STABILITY.md](../../API_STABILITY.md)

### OSS delivery posture

- Aligns installation examples and release guidance with the `3.0.1` line
- Keeps benchmark governance as diff-only, with benchmark output treated as observational
- Continues to validate docs, consumer smoke, and release automation in CI

### Runtime and operational model

- Deterministic request encoding and explicit decoding strategies remain the default behavior
- Event delivery keeps bounded buffering, listener isolation, and telemetry
- Download persistence continues to use append-log durability
- WebSocket reconnect behavior keeps handshake-aware classification and suppression rules

## Upgrade Notes

- Package installation instructions now target `3.0.1`
- The latest supported public release line is `3.x`, starting from `3.0.1`
- Internal policy layers such as `RequestEncodingPolicy`, `ResponseDecodingStrategy`, reconnect taxonomy internals, and persistence file formats remain non-contract implementation details

## Validation

Release validation for 3.0.1 includes:

- `swift test`
- docs / contract sync
- documentation smoke build and run
- benchmark target smoke build and quick benchmark run
- external consumer smoke build
