# Public API Surface Allowlists

Each `*.allowlist` file is the canonical inventory of `public` declarations
exported from one product. The lists are generated by
`Scripts/collect_public_symbols.py` from the Swift symbol-graph dump and
validated by `Scripts/check_docs_contract_sync.sh`, so any drift between
the source code and these files fails CI.

## Current sizes (2026-05)

| Product | Public declarations |
|---|---:|
| `InnoNetwork` (core) | 867 |
| `InnoNetworkWebSocket` | 172 |
| `InnoNetworkDownload` | 136 |
| `InnoNetworkTestSupport` | 76 |
| `InnoNetworkPersistentCache` | 47 |
| `InnoNetworkOpenAPI` | 19 |
| `InnoNetworkTrust` | 13 |
| **Total** | **1,330** |

## Why this matters

For a single-maintainer client-side Swift networking library, 1,330 public
declarations is unusually large — roughly 4× `Get`'s surface and ~10× the
`URLSession`-only "two functions and a `Decoder`" baseline.

Every public symbol is:

- A regression-testing obligation in CI (the allowlists make this concrete).
- A backwards-compatibility promise for the lifetime of the surrounding
  contract tier (Stable / Provisionally Stable, see `API_STABILITY.md`).
- A documentation obligation — DocC catalogues are spot-checked by
  `check_docs_contract_sync.sh`.

The most common reason a symbol ends up here without serving the public
contract is an internal helper that needs to cross the compilation-target
boundary inside the package; those should be expressed with `package`
visibility, not `public`.

## Reduction roadmap

The library targets `core` ~300 and total ~600 over the 4.x line. The
sweep is **scoped to a follow-up PR** because each demotion needs a
dependency-graph audit (which test target depends on the symbol, which
DocC link references it, whether any consumer recipe surfaces it).

The roadmap is intentionally not landed here; allowlist edits change the
public contract, and a measured, reviewable batch is safer than a single
sweep across all seven runtime products.

## Working with the allowlists

```bash
# Regenerate from the current symbol-graph dump
swift build --target InnoNetwork -Xswiftc -emit-symbol-graph \
  -Xswiftc -emit-symbol-graph-dir -Xswiftc .build/symbolgraph
python3 Scripts/collect_public_symbols.py .

# Validate the allowlists match the symbol-graph dump
bash Scripts/check_docs_contract_sync.sh
```

When intentionally adding a new public declaration, run the regenerate step
and commit the diff to the appropriate allowlist together with the source
change so the contract gate stays green in CI.
